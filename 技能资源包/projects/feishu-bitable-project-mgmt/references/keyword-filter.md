```

│                   消息输入                           │
└─────────────────┬───────────────────────────────────┘
                  ▼
┌─────────────────────────────────────────────────────┐
│                 基础层（硬性规则）                    │
│  上下文绑定：关键词 + 动词/业务字段                    │
│  误判率目标：<5%                                    │
└─────────────────┬───────────────────────────────────┘
                  ▼
        ┌─────────────────┐
        │    匹配成功？     │
        └────────┬────────┘
                 │
       ┌─────────┴─────────┐
       ▼                   ▼
      是                   否
       │                   │
       ▼                   ▼
┌──────────────┐    ┌─────────────────────────────────┐
│ 语义层验证   │    │        进入下一规则检查           │

└──────┬───────┘

└─────────────────────────────────────────────────────┘
```

## 基础层设计

### 硬性规则设计原则

**必须结合业务上下文**，避免纯关键词误判：

| 场景 | 错误触发 | 正确触发 |
|-----|---------|---------|
| 法务 | 合同（合同法学习） | 合同+审核/签署/条款 |
| 销售 | 合同（合同法学习） | 合同+金额/客户/回款 |
| 技术 | 接口（接口文档） | 接口+调用/对接/数据 |
| 运营 | 活动（促销活动） | 活动+参与/转化/ROI |

### 基础层配置示例

```formula
// 法务场景规则
法务触发 = AND(
  CONTAINS(消息, {"合同", "协议", "条款"}),
  OR(
    CONTAINS(消息, {"审核", "签署", "审批", "法务"}),
    CONTAINS(消息, {"金额", "日期", "甲方", "乙方"})
  )
)

// 销售场景规则
销售触发 = AND(
  CONTAINS(消息, {"客户", "订单", "合同"}),
  OR(
    CONTAINS(消息, {"金额", "回款", "成交"}),
    CONTAINS(消息, {"跟进", "报价", "方案"})
  )
)

// 技术场景规则
技术触发 = AND(
  CONTAINS(消息, {"接口", "API", "服务"}),
  OR(
    CONTAINS(消息, {"调用", "对接", "数据"}),
    CONTAINS(消息, {"开发", "测试", "部署"})
  )
)
```

### 部门差异化权重

```formula
// 同一关键词在不同部门的权重
方案权重 = SWITCH(
  当前部门,
  "技术部", 0.2,   // 技术部：方案指技术方案，高频正常
  "市场部", 0.8,   // 市场部：方案指营销方案，需关注
  "销售部", 0.5,
  0.3               // 默认低权重
)

// 工单权重
工单权重 = SWITCH(
  当前部门,
  "运维部", 0.9,   // 运维部：工单=故障单，高优先级
  "行政部", 0.3,   // 行政部：工单=日常流程
  0.5
)
```

## 语义层设计

### AI验证提示词模板

```prompt
判断消息是否属于[{场景}]场景：

规则：需包含{核心要素}

消息内容：{text}

输出格式：
- 是否匹配：是/否
- 置信度：0-100
- 匹配理由：{关键发现}
```

### 场景验证示例

**法务合同审核场景**：
```prompt
判断消息是否属于[法务合同审核]场景：

规则：需包含合同主体、责任条款、法律效力描述

消息内容：{text}

输出格式：
- 是否匹配：是/否
- 置信度：0-100
- 匹配理由：{关键发现}
```

**销售跟进场景**：
```prompt
判断消息是否属于[销售客户跟进]场景：

规则：需包含客户名称、跟进动作、预期结果

消息内容：{text}

输出格式：
- 是否匹配：是/否
- 置信度：0-100
- 匹配理由：{关键发现}
```

### 置信度处理决策

| 置信度范围 | 处理方式 | 说明 |
|-----------|---------|------|
| >85% | 自动处理 | 高置信度，直接执行 |
| 30-85% | 降低权重+复核 | 中置信度，加入复核队列 |
| <30% | 白名单过滤 | 低置信度，视为误判 |

```formula
// 置信度处理决策
处理方式 = SWITCH(
  TRUE(),
  置信度 > 85, "自动处理",
  置信度 >= 30, "降低权重+人工复核",
  "加入白名单"
)

// 权重调整
最终权重 = IF(置信度 >= 30, 基础权重 * (置信度/100), 0)
```

## 实施配置

### 飞书多维表格配置

**字段设计**：

| 字段名 | 类型 | 公式 | 说明 |
|-------|------|------|------|
| 基础匹配 | 单选 | IF(基础层规则, "匹配", "未匹配") | 硬性规则结果 |
| 语义置信度 | 数字 | AI计算 | 0-100分 |
| 最终决策 | 单选 | 根据置信度判断 | 自动/复核/过滤 |
| 处理状态 | 单选 | 待处理/已处理/已过滤 | 流程状态 |

**自动化规则**：

```formula
// 规则1：基础层匹配
触发条件：消息创建
动作：执行基础层规则，更新"基础匹配"字段

// 规则2：语义层验证
触发条件：基础匹配="匹配"
动作：调用AI验证，更新"语义置信度"字段

// 规则3：自动处理
触发条件：语义置信度 > 85
动作：更新"处理状态"="已处理"，发送通知

// 规则4：人工复核
触发条件：语义置信度 BETWEEN(30, 85)
动作：更新"处理状态"="待处理"，加入复核队列
```

### 场景化配置清单

| 场景 | 核心关键词 | 关联字段 | 权重基准 |
|-----|----------|---------|---------|
| 法务 | 合同/协议/条款 | 审核/签署/法务 | 0.9 |
| 销售 | 客户/订单/金额 | 跟进/回款/报价 | 0.8 |
| 技术 | 接口/API/服务 | 调用/开发/部署 | 0.7 |
| 运营 | 活动/用户/转化 | 参与/拉新/ROI | 0.6 |
| 行政 | 工单/审批/流程 | 发起/处理/完成 | 0.4 |

### 效果评估指标

| 指标 | 目标值 | 测量方法 |
|-----|-------|---------|
| 误判率 | <5% | 人工复核发现误判数/总处理数 |
| 召回率 | >90% | 真实匹配数/应匹配总数 |
| 处理速度 | <2秒 | 从消息到决策的时间 |
| 自动化率 | >80% | 自动处理数/总处理数 |
