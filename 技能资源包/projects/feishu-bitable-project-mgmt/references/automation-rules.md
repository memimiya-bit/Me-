## 概述

自动化规则是飞书多维表格 AI 能力的核心，通过配置触发条件和执行动作，实现字段自动计算、状态更新、消息通知等功能。

**规则设计原则**：
- 触发条件应具体明确
- 动作执行应可逆或有日志
- 避免规则之间冲突
- 定期检查规则执行效果

## 智能状态计算

### 规则说明

根据项目的时间、进度信息自动计算当前状态，无需手动更新。

### 配置路径

「高级设置」→「自动化规则」→「新建规则」

### 触发条件

**触发时机**：当以下任一字段变更时
- 计划结束日期
- 完成度
- 项目阶段

### 执行动作

```公式
IF(完成度=100, "已完成",
  IF(AND(计划结束日期<TODAY(), 完成度<100), "已延期",
    IF(完成度>0, "进行中", "待启动")))
```

### 状态判断逻辑

```
                    ┌─────────────┐
                    │  完成度=100 │
                    └──────┬──────┘
                           │ Yes
                    ┌──────▼──────┐
                    │  已完成     │
                    └─────────────┘
                           │ No
                           ▼
                    ┌─────────────┐
                    │结束日期<Today│
                    │ AND 完成度<100│
                    └──────┬──────┘
                           │ Yes
                    ┌──────▼──────┐
                    │  已延期     │
                    └─────────────┘
                           │ No
                           ▼
                    ┌─────────────┐
                    │  完成度>0   │
                    └──────┬──────┘
                      Yes  │  No
                    ┌──────▼──────┐
                    │  进行中     │
                    └─────────────┘
                           │
                           ▼
                    ┌─────────────┐
                    │  待启动     │
                    └─────────────┘
```

## 风险等级自动计算

### 规则说明

综合延期天数和预算超支率两个维度，自动计算项目风险等级。

### 触发条件

**触发时机**：当以下任一字段变更时
- 计划结束日期
- 预算金额
- 实际花费

### 计算公式

```公式
延期天数 = MAX(0, TODAY() - 计划结束日期)
预算超支率 = IF(预算金额>0, (实际花费-预算金额)/预算金额*100, 0)

IF(延期天数>7 OR 预算超支率>20, "高风险",
  IF(延期天数>=3 OR 预算超支率>=10, "中风险",
    IF(延期天数>0 OR 预算超支率>0, "低风险",
      "无风险")))
```

### 风险等级定义

| 风险等级 | 延期天数 | 预算超支率 | 颜色标识 | 处理建议 |
|---------|---------|-----------|---------|---------|
| 高风险 | >7天 | >20% | 红色 | 立即处理，升级汇报 |
| 中风险 | 3-7天 | 10-20% | 橙色 | 重点关注，制定对策 |
| 低风险 | 1-2天 | 1-10% | 黄色 | 保持监控 |
| 无风险 | 0天 | 0% | 绿色 | 正常推进 |

### 风险计算示例

| 场景 | 计划结束日期 | 当前日期 | 预算金额 | 实际花费 | 风险等级 |
|-----|------------|---------|---------|---------|---------|
| 正常项目 | 2024-12-31 | 2024-11-15 | 100,000 | 60,000 | 无风险 |
| 轻度延期 | 2024-11-10 | 2024-11-15 | 100,000 | 80,000 | 低风险 |
| 中度延期 | 2024-11-12 | 2024-11-15 | 100,000 | 85,000 | 中风险 |
| 严重延期 | 2024-11-01 | 2024-11-15 | 100,000 | 120,000 | 高风险 |
| 超支严重 | 2024-12-31 | 2024-11-15 | 100,000 | 130,000 | 高风险 |

## 下次沟通日期计算

### 规则说明

根据设定的沟通频率，自动计算下次沟通的计划日期。

### 触发条件

**触发时机**：当「沟通频率」字段变更时

### 计算公式

```公式
IF(沟通频率="每日", TODAY(),
  IF(沟通频率="每周", TODAY()+7,
    IF(沟通频率="每两周", TODAY()+14,
      IF(沟通频率="每月", EDATE(TODAY(),1), ""))))
```

### 日期计算规则

| 沟通频率 | 计算方式 | 示例（假设今天为11月15日） |
|---------|---------|------------------------|
| 每日 | TODAY() | 11月15日 |
| 每周 | TODAY()+7 | 11月22日 |
| 每两周 | TODAY()+14 | 11月29日 |
| 每月 | EDATE(TODAY(),1) | 12月15日 |
| 按需 | 空值 | - |

## 逾期提醒工作流

### 规则说明

每天自动检查逾期项目，发送提醒消息并更新状态。

### 配置路径

「高级设置」→「自动化规则」→「定时触发」

### 触发条件

**触发类型**：定时触发
**执行时间**：每天上午 9:00

### 筛选条件

```
计划结束日期 < TODAY() AND 状态 ≠ "已完成"
```

### 执行动作

#### 动作一：更新状态

```
字段：状态
值："已延期"
```

#### 动作二：发送飞书消息

**消息模板**：
```
【项目逾期提醒】

项目名称：{项目名称}
计划结束：{计划结束日期}
当前完成度：{完成度}%
逾期天数：{延期天数} 天

请尽快处理，如有延期原因请在项目中备注。
```

#### 动作三：@项目负责人

```
人员：{项目负责人}
```

### 完整工作流配置

```
触发器：每天 09:00 执行

条件筛选：
  计划结束日期 < TODAY()
  AND 状态 ≠ "已完成"

执行动作：
  1. 更新字段「状态」为「已延期」
  2. 发送消息给「项目负责人」
  3. 在记录中新增备注：系统自动标记逾期
```

## 新项目创建通知

### 规则说明

当创建新项目时，自动通知主责部门负责人。

### 触发条件

**触发时机**：当记录创建时

### 筛选条件

```
项目名称 ≠ ""
```

### 执行动作

#### 动作一：自动分配项目ID

```
字段：项目ID
值：CONCATENATE("PRJ-", TEXT(ROW()-1, "000"))
```

#### 动作二：设置创建时间

```
字段：创建时间
值：NOW()
```

#### 动作三：发送通知

**消息模板**：
```
【新项目创建通知】

项目名称：{项目名称}
项目类型：{项目类型}
对接模式：{对接模式}
主责部门：{主责部门}
项目负责人：{项目负责人}

请登录飞书多维表格查看详情。
```

## 阶段变更记录

### 规则说明

当项目阶段变更时，自动记录变更历史，并在特定阶段触发相关操作。

### 触发条件

**触发时机**：当「项目阶段」字段变更时

### 执行动作

#### 动作一：记录变更历史

```
字段：变更记录
值：CONCATENATE(
  变更记录,
  CHAR(10),
  TEXT(TODAY(), "YYYY-MM-DD"),
  " | ",
  "阶段变更：",
  {旧值},
  " → ",
  {新值},
  "（变更人：",
  CURRENT_USER(),
  "）"
)
```

#### 动作二：特定阶段特殊处理

**当阶段变更为「验收」时**：
```
动作：自动@质量负责人
消息：项目「{项目名称}」已进入验收阶段，请安排质量验收。
```

**当阶段变更为「完成」时**：
```
动作：设置实际完成日期为今天
设置完成度为100%
消息：恭喜！项目「{项目名称}」已完成验收。
```

## 规则执行优先级

当多个规则可能同时触发时，按以下优先级执行：

1. **数据验证规则**（最高）
2. **智能状态计算**
3. **风险等级计算**
4. **沟通日期计算**
5. **工作流通知规则**

**冲突处理**：
- 同一字段的后续规则覆盖前面的结果
- 通知类规则互不干扰，可同时执行
