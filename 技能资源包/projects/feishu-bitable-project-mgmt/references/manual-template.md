## 模板库结构

### 飞书多维表格「操作模板库」表

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 场景 | 单行文本 | 业务场景名称 |
| 触发条件 | 多行文本 | 消息匹配规则 |
| 操作步骤 | 多行文本 | 自然语言描述+{字段}占位 |
| 关联数据 | 多行文本 | 提取字段列表 |
| 适用部门 | 多选 | 使用该模板的部门 |
| 状态 | 单选 | 启用/禁用/测试中 |
| 创建时间 | 日期 | 自动记录 |

---

## 模板设计规范

### 设计原则

1. **操作步骤用自然语言描述**
   - 正确："在OA系统提交ID为{项目ID}的合同审核"
   - 错误："submitContractReview(projectId)"

2. **关联数据用{字段名}占位**
   - 占位符格式：`{字段名}`
   - 字段名与消息中的提取规则对应

3. **步骤必须可执行**
   - 每步对应一个具体动作
   - 避免模糊描述

## 配置示例

### 示例1：合同审核

```
场景：合同审核
触发条件：消息包含"合同" AND 消息包含"法务"
操作步骤：
  1. 在OA系统创建合同审核任务，关联项目ID为{项目ID}
  2. 关联合同版本号{合同版本}
  3. 通知法务负责人{法务负责人}
  4. 设置审核截止日期为{截止日期}
关联数据：项目ID、合同版本、法务负责人、截止日期
适用部门：法务部、销售部、采购部
状态：启用
```

### 示例2：库存预警

```
场景：库存预警
触发条件：库存 < 安全阈值
操作步骤：
  1. 生成采购申请单，关联项目ID为{项目ID}
  2. 计算补货数量：当前库存{当前库存}，安全库存{安全库存}
  3. 通知采购负责人{采购负责人}
  4. 在采购系统提交采购申请
关联数据：项目ID、当前库存、安全库存、采购负责人
适用部门：仓储部、采购部
状态：启用
```

### 示例3：客户投诉处理

```
场景：客户投诉处理
触发条件：消息包含"投诉" OR 消息包含"问题反馈"
操作步骤：
  1. 在CRM系统创建投诉工单，客户名称为{客户名称}
  2. 关联相关订单号{订单号}
  3. 分配给客服负责人{客服负责人}
  4. 设置处理时限为{处理时限}小时
  5. 发送投诉确认通知给客户
关联数据：客户名称、订单号、客服负责人、处理时限
适用部门：客服部、运营部
状态：启用
```

### 示例4：项目延期预警

```
场景：项目延期预警
触发条件：计划完成日期 < TODAY() AND 状态 ≠ "已完成"
操作步骤：
  1. 在项目管理系统标记项目{项目ID}为延期状态
  2. 生成延期分析报告，包含：
     - 原计划完成日期：{原计划日期}
     - 当前进度：{完成度}%
     - 延期原因：{延期原因}
  3. 通知项目经理{项目经理}和部门负责人{负责人}
  4. 创建延期复盘任务，设置评审日期
关联数据：项目ID、原计划日期、完成度、延期原因、项目经理、负责人
适用部门：项目部、各业务部门
状态：启用
```

## 自动化规则配置

### 飞书自动化规则设置

```
触发器：当收到群消息

条件配置：
  条件1：消息包含"合同"
  条件2：消息包含"法务"
  逻辑：条件1 AND 条件2

动作配置：
  1. 搜索「操作模板库」
     - 筛选：场景 = "合同审核"
     - 筛选：状态 = "启用"
  
  2. 从消息中提取字段
     - 项目ID：正则匹配 "项目[：:]?\s*([A-Z]\w+)"
     - 合同版本：正则匹配 "版本[：:]?\s*([\d.]+)"
  
  3. 生成可执行指令
     - 模板：{操作步骤}
     - 替换：{项目ID} → 提取值A
     - 替换：{合同版本} → 提取值B
  
  4. 发送指令到待办
     - 接收人：当前用户
     - 内容：生成的指令
     - 确认按钮：执行 / 修正
  
  5. 记录执行日志
     - 模板名称
     - 提取字段
     - 生成时间
```

## 规则即文档原则

业务人员可直接修改「操作模板库」表，无需懂技术：

- 修改触发条件：调整消息匹配规则
- 修改操作步骤：用自然语言描述动作
- 新增模板：复制现有模板，修改对应字段
- 禁用模板：将状态改为"禁用"
