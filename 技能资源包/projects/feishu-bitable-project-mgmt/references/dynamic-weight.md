## 权重计算模型

### 综合权重公式

```formula
动态权重 = (
  {岗位权重} * 0.3 + 
  {项目权重} * 0.35 + 
  {历史权重} * 0.25 + 
  {时效权重} * 0.1
)
```

### 权重维度说明

| 维度 | 权重 | 数据来源 |
|-----|------|---------|
| 岗位权重 | 30% | 岗位-词汇矩阵 |
| 项目权重 | 35% | 历史项目经验 |
| 历史权重 | 25% | 用户反馈数据 |
| 时效权重 | 10% | 业务变化信号 |

## 岗位维度权重

### 岗位-词汇矩阵

```yaml
词汇: 接口

岗位权重配置:
  技术部:
    权重: 0.9
    含义: "API接口（系统对接）"
    上下文: "调用/开发/测试/部署"
    
  销售部:
    权重: 0.4
    含义: "客户对接人（人际沟通）"
    上下文: "联系/拜访/沟通"
    
  运维部:
    权重: 0.7
    含义: "服务接口（监控指标）"
    上下文: "调用量/响应时间/可用性"
```

### 动态调整示例

```formula
// 技术总监在"架构评审"项目中多次忽略"接口"提醒
// 但在"系统集成"项目中100%需要处理

// 系统自动记录项目上下文差异
项目上下文差异 = {
  "架构评审项目": {
    "忽略次数": 5,
    "关键词": "接口设计/接口规范",
    "建议权重": 0.3
  },
  "系统集成项目": {
    "处理次数": 20,
    "关键词": "接口对接/接口调试",
    "建议权重": 0.95
  }
}

// 为该词创建项目维度权重规则
新规则 = IF(
  项目类型 = "系统集成",
  0.95,
  IF(
    项目类型 = "架构评审",
    0.3,
    0.5  # 默认权重
  )
)
```

## 项目维度权重

### 权重计算逻辑

```formula
项目权重 = AI_CALCULATE_WEIGHT(
  "基于{岗位角色}+{历史行为}+{当前项目}，动态计算术语权重：
   
   历史准确率：该词汇在该岗位的触发准确率
   业务重要性：词汇关联的业务流程关键度
   时效敏感度：词汇对应的业务时效要求
   协作影响度：词汇涉及的跨部门协作范围
   
  输出：0-10分权重 + 更新建议"
)
```

### 项目相似度匹配

```formula
// 查找相似历史项目
相似项目 = FILTER(
  历史项目库,
  AND(
    项目类型 ≈ 当前项目类型,
    行业 ≈ 当前行业,
    规模 ≈ 当前规模
  )
)

// 从相似项目学习权重
学到的权重 = AVERAGE(
  相似项目.词汇权重
)
```

### 根因分析与修复

```formula
// 根因分类
根因 = SWITCH(
  误判类型,
  "上下文缺失", "补充{上下文规则}",
  "语义理解错误", "优化{语义模型}",
  "权重配置错误", "调整{权重系数}",
  "未知原因"
)

// 修复方案生成
修复方案 = AI_GENERATE_FIX_PLAN(
  "分析'{term}'误判模式，建议：
   调整权重（当前:{current_weight}→建议:{new_weight}）
   补充上下文规则（示例：{negative_example}不应触发）
   拆分术语（如'接口'拆分为'API接口'和'客户接口'）
   暂时禁用直到规则完善"
)
```

## 实施配置

### 飞书多维表格字段

| 字段名 | 类型 | 公式 |
|-------|------|------|
| 岗位权重 | 数字 | 查表获取 |
| 项目权重 | 数字 | 相似项目平均 |
| 历史权重 | 数字 | 反馈数据计算 |
| 综合权重 | 数字 | 加权求和 |
| 权重状态 | 单选 | 稳定/学习中/待优化 |

### 自动化规则

```formula
// 权重自动计算
综合权重 = 岗位权重*0.3 + 项目权重*0.35 + 历史权重*0.25 + 时效权重*0.1

// 学习触发
IF(误判反馈 OR 用户忽略, 触发学习流程)

// 权重更新
新权重 = IF(
  学习样本数 > 20,
  综合权重 * 0.8 + 新反馈权重 * 0.2,
  综合权重
)
```

### 效果监控

```formula
// 权重健康度
权重健康度 = IF(
  误判率 < 10%, "健康",
  误判率 < 30%, "学习中",
  "需优化"
)

// 学习进度
学习进度 = 已收集样本数 / 目标样本数 * 100%
```
