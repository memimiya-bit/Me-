## 核心理念

**痛点**：70%项目经理手动拆解任务，新项目平均耗时8.3小时，且忽略部门历史经验。

**目标**：让AI基于历史数据自动生成符合部门工作习惯的任务结构。

## 任务模式挖掘

### 历史数据分析

```formula
部门任务模板 = AI_ANALYZE(
  {历史项目数据}, 
  "提取{部门名称}在{项目类型}中的高频任务模式，按阶段聚合"
)
```

### 模式识别要素

| 要素 | 说明 | 提取方式 |
|-----|------|---------|
| 阶段划分 | 部门习惯的项目阶段 | 历史项目阶段分布 |
| 任务类型 | 高频任务分类 | 任务名称聚类 |
| 持续时间 | 各任务的典型工期 | 历史工时统计 |
| 依赖关系 | 任务间的前后置 | 任务时序分析 |
| 成功率 | 各任务的完成率 | 历史完成率统计 |
| 资源需求 | 常用资源配置 | 历史资源统计 |

### 模式评估指标

```formula
任务模式置信度 = (
  {出现频次} / {总项目数} × 0.3 +
  {平均成功率} × 0.4 +
  {最近更新时间}权重 × 0.3
)
```

## 智能任务树生成

### 生成公式

```formula
建议任务树 = AI_GENERATE(
  "基于{部门任务模板}和{当前项目目标}，生成符合{部门工作习惯}的任务结构，
   包含责任矩阵(RACI)，标注各任务的历史成功率和参考案例"
)
```

### 任务树结构

```json
{
  "project": "{项目名称}",
  "department": "{部门名称}",
  "phase": [
    {
      "name": "{阶段名称}",
      "owner": "{负责人}",
      "tasks": [
        {
          "name": "{任务名称}",
          "duration": "{预计工期}",
          "history_success_rate": 0.85,
          "reference_case": "{参考案例}",
          "dependencies": ["{前置任务}"],
          "risks": ["{风险点}"]
        }
      ]
    }
  ],
  "milestones": [
    {
      "name": "{里程碑名称}",
      "date": "{计划日期}",
      "criteria": "{交付标准}"
    }
  ]
}
```

### 生成质量评估

```formula
任务树完整性 = (
  模板覆盖率 × 0.4 +
  历史借鉴度 × 0.3 +
  依赖完整性 × 0.2 +
  风险识别度 × 0.1
)
```

## 模板库管理

### 模板分类

| 类别 | 说明 | 示例 |
|-----|------|-----|
| 项目类型模板 | 按业务类型分类 | 电商活动/功能开发/市场推广 |
| 部门模板 | 按部门习惯分类 | 运营部/研发部/市场部 |
| 阶段模板 | 按项目阶段分类 | 启动/规划/执行/收尾 |
| 任务模板 | 通用任务模式 | 需求评审/代码Review |

### 模板更新机制

```formula
模板新鲜度 = IF(
  最后更新 > TODAY() - 90天,
  "活跃模板（近期项目验证）",
  "待验证模板（建议参考）"
)

更新建议 = AI_ANALYZE(
  {新项目实践} + {旧模板偏差},
  "提取可优化的任务模式，更新至模板库"
)
```

### 模板质量评分

```formula
模板质量 = (
  使用次数 × 0.2 +
  平均成功率 × 0.5 +
  用户评分 × 0.3
)
```

### 模板生命周期

```
创建 → 验证(3个项目) → 正式发布 → 监控使用 → {效果下降?} → 优化/归档
                                                    ↓
                                              持续优化
```

| 指标 | 说明 |
|-----|------|
