## 资源冲突检测模型

### 冲突指数计算

**基础冲突指数**：
```公式
资源冲突指数 = (COUNTIFS(资源分配表::负责人, @当前资源, 资源分配表::计划开始日期, "<="&TODAY()+7) / 最大负荷) × 100
```

**扩展冲突指数（含结束日期）**：
```公式
资源冲突指数 = (
  COUNTIFS(
    负责人, @资源,
    计划开始日期, "<="&TODAY()+7,
    计划结束日期, ">="&TODAY()
  ) / 最大负荷
) × 100
```

### 冲突等级判定

```公式
冲突等级 = SWITCH(TRUE(),
  冲突指数 > 120, "🔴 严重冲突",
  冲突指数 > 100, "🟠 高度冲突",
  冲突指数 > 80, "🟡 中度冲突",
  "✅ 正常"
)
```

### 多维度冲突检测

**时间维度冲突**：
```公式
时间冲突数 = SUMPRODUCT((任务开始<=TODAY()+7)*(任务结束>=TODAY())*(负责人=@资源))
```

**技能维度冲突**：
```公式
技能冲突 = IF(技能需求数 > 具备该技能人数, "技能不足", "技能充足")
```

**优先级维度冲突**：
```公式
高优先级冲突 = COUNTIFS(负责人, @资源, 优先级, "紧急", 日期范围, "<="&TODAY()+7)
```

## 智能调度算法

### 调度优先级计算

**多因素加权公式**：
```公式
调度优先级 = (
  项目重要性 × 0.3 + 
  任务紧急度 × 0.25 + 
  资源技能匹配度 × 0.2 + 
  成本影响度 × 0.15 + 
  客户满意度影响 × 0.1
)
```

**各因素计算方式**：

| 因素 | 计算公式 | 权重 |
|-----|---------|-----|
| 项目重要性 | RANK(项目价值, DESC) / 总项目数 × 100 | 30% |
| 任务紧急度 | (总工期 - 剩余工期) / 总工期 × 100 | 25% |
| 技能匹配度 | 匹配技能数 / 需求技能数 × 100 | 20% |
| 成本影响度 | 任务预算 / 项目总预算 × 100 | 15% |
| 客户影响 | 客户等级系数 × 满意度系数 | 10% |

### 资源最优分配

**最优资源推荐**：
```公式
最优资源 = INDEX(
  资源列表,
  MATCH(
    MAX(
      资源技能匹配度×0.4 + 
      资源可用性×0.3 + 
      历史绩效×0.2 + 
      成本系数×0.1
    ),
    综合评分,
    0
  )
)
```

**综合评分计算**：
```公式
综合评分 = 
  (已匹配技能数/需求技能数) × 0.4 +
  (可用时间/需求时间) × 0.3 +
  (历史按时完成率) × 0.2 +
  (成本效率) × 0.1
```

### 智能调度建议

**调度建议生成**：
```公式
调度建议 = IF(冲突等级 ≠ "✅ 正常",
  AI_GENERATE(
    "资源[" & @资源 & "]在" & TEXT(TODAY()+7,"yyyy-mm-dd") & "期间冲突指数为" & 冲突指数 & "%。" &
    "高优先级任务：" & 高优先级任务列表 & "。" &
    "请按以下格式提供建议：1.调整任务时间 2.增加资源 3.重新分配任务 4.优先级调整"
  ),
  "无需调度，当前负载正常"
)
```

// 周负载计算

## 冲突解决策略

### 冲突解决优先级

```公式
冲突解决优先级 = (
  项目价值 × 0.3 + 
  客户等级 × 0.25 + 
  延期影响 × 0.2 + 
  资源稀缺度 × 0.15 + 
  成本影响 × 0.1
)
```

**各因素计算**：

| 因素 | 计算方式 |
|-----|---------|
| 项目价值 | 项目合同金额 / Portfolio总金额 |
| 客户等级 | VIP=100, 重要=75, 普通=50 |
| 延期影响 | 延期导致损失 / 项目总金额 |
| 资源稀缺度 | 1 - (具备该技能人数 / 总需求人数) |
| 成本影响 | 任务预算 / 项目总预算 |

### 自动调整建议

```公式
调整建议 = SWITCH(冲突等级,
  "🔴 严重冲突", "建议延后低优先级任务" & 高优先级任务 & "，或增加资源",
  "🟠 高度冲突", "建议与" & 冲突任务负责人 & "协调调整时间",
  "🟡 中度冲突", "建议监控，等待资源释放",
  "✅ 正常", "无需调整"
)
```

### 手动调整触发条件

```公式
需要人工介入 = OR(
  冲突指数 > 150,
  冲突持续天数 > 5,
  涉及多个项目,
  涉及高价值客户
)
```

## 调度优化模板

### 调度分析仪表盘

**卡片配置**：

| 卡片 | 指标 | 公式 | 目标值 |
|-----|------|------|-------|
| 资源利用率 | 实际使用/总能力 | SUM(实际工时)/SUM(可用工时) | 75-85% |
| 冲突发生率 | 冲突次数/总分配 | COUNTIFS(冲突等级,"🔴")/COUNT(资源) | <10% |
| 调度响应时间 | 冲突到解决 | 平均(解决时间-发现时间) | <24h |
| 资源均衡度 | 负载标准差 | STD(各资源负载) | <15% |
