## 自动化规则概述

### 自动化能力层级

| 层级 | 功能 | 成本 | 说明 |
|-----|------|-----|------|
| 基础自动化 | 字段更新、消息通知 | 免费 | 基础触发条件 |
| 高级自动化 | 跨表同步、条件分支 | 免费 | 进阶逻辑 |
| AI自动化 | 智能分析、预测 | 消耗点数 | 需AI能力 |

### 规则设计原则

1. **单一职责**：每个规则只做一件事
2. **触发明确**：条件判断清晰无歧义
3. **可追溯**：记录自动化执行日志
4. **容错处理**：异常情况有兜底方案

## 智能提醒自动化

### 规则1：任务到期提醒

**配置**：
```
触发时机：当 {计划完成日期} = TODAY()+1 且 {任务状态} = "进行中"
执行动作：发送飞书消息给 {负责人}
消息模板：
  任务【{任务名称}】明天到期
  当前进度：{完成度}%
  请及时处理！
```

**高级版（含AI）**：
```
触发时机：当 {计划完成日期} = TODAY()+3
执行动作：
  1. AI分析当前进度是否达标
  2. 生成赶工建议
  3. 发送个性化提醒
消息模板：
  【智能提醒】{任务名称}
  AI分析：{AI分析结果}
  建议：{AI建议}
```

### 规则2：延期预警

**配置**：
```
触发时机：当 {计划完成日期} < TODAY() 且 {任务状态} ≠ "已完成"
执行动作：
  1. 更新 {状态} = "已延期"
  2. 更新 {延期天数} = TODAY()-{计划完成日期}
  3. 发送消息给 {负责人} 和 {上级}
消息模板：
  【延期提醒】{任务名称}
  已延期 {延期天数} 天
  请说明延期原因并制定赶工计划
```

### 规则3：超时未响应

**配置**：
```
触发时机：每天检查，当 {最后跟进日期} < TODAY()-3 且 {任务状态} = "进行中"
执行动作：发送提醒给 {负责人}
消息模板：
  【待跟进】您有任务已超过3天未跟进
  {任务名称} - {客户/项目}
```

### 规则4：里程碑达成

**配置**：
```
触发时机：当 {完成度} = 100% 且 {之前完成度} < 100%
执行动作：
  1. 更新 {任务状态} = "已完成"
  2. 更新 {实际完成日期} = TODAY()
  3. 发送喜报给 {项目组}
消息模板：
  恭喜！{任务名称} 已完成
  用时：{实际完成日期}-{创建日期} 天
  负责人：{负责人}
```

## 数据同步自动化

### 规则1：跨表数据汇总

**场景**：生产日报表 → 项目进度表

**配置**：
```
触发时机：当 {生产日报表} 有新记录
执行动作：更新 {项目进度表}
更新条件：{订单编号} = {生产日报表.订单编号}
更新字段：
  {今日完成量} = {今日完成量} + {生产日报表.产量}
  {整体进度} = {累计完成量} / {总订单量} × 100%
```

### 规则2：状态同步

**场景**：订单状态变更 → 通知相关部门

**配置**：
```
触发时机：当 {订单状态} 变更
执行动作：
  1. 根据新状态通知对应部门
  2. 更新相关联的 {任务状态}
通知规则：
  "已确认" → 通知生产部
  "生产中" → 通知品质部
  "已发货" → 通知客户
  "已完成" → 通知财务部
```

### 规则3：库存预警联动

**场景**：库存低于安全库存 → 自动生成采购申请

**配置**：
```
触发时机：当 {库存数量} < {安全库存} 且 {采购状态} = "无需采购"
执行动作：
  1. 创建新记录到 {采购申请表}
  2. 填写 {商品名称}、{缺少数量}、{紧急程度}
  3. 通知采购负责人
```

### 规则4：人员变动同步

**场景**：人员调岗 → 更新所有相关任务

**配置**：
```
触发时机：当 {员工.部门} 变更
执行动作：对所有 {负责人} = {员工} 的记录
更新字段：
  {部门} = {新部门}
  {上级} = {新上级}
```

## 状态更新自动化

### 规则1：自动计算状态

**公式配置**：
```公式
状态 = IF({完成度}=100, "已完成",
  IF({计划完成日期}<TODAY() AND {完成度}<100, "已延期",
    IF({完成度}>0, "进行中", "待启动")))
```

### 规则2：风险等级自动更新

**公式配置**：
```公式
风险评分 = ({延期天数}×0.4) + ({成本超支率}×0.3) + ({质量缺陷数}×0.3)

风险等级 = IF({风险评分}≥7, "高风险",
  IF({风险评分}≥4, "中风险", "低风险"))
```

### 规则3：完成度自动计算

**公式配置**：
```公式
完成度 = IF({工序总数}>0, 
  SUM({各工序完成情况}) / {工序总数} × 100%,
  IF({阶段}="规划中", 0,
    IF({阶段}="执行中", 50,
      IF({阶段}="验收中", 90, 100))))
```

### 规则4：到期提醒状态

**配置**：
```
触发时机：每天检查
条件：{计划完成日期} - TODAY() ≤ 7 且 {状态} = "进行中"
执行动作：
  1. 更新 {提醒状态} = "即将到期"
  2. 设置条件格式（黄色高亮）
```

## 审批流程自动化

### 规则1：预算审批

**配置**：
```
触发时机：当 {申请金额} > 10000
执行动作：
  1. 创建审批记录
  2. 通知 {审批人}
  3. 等待审批结果
审批通过后：
  更新 {预算状态} = "已批准"
审批拒绝后：
  更新 {预算状态} = "已拒绝"
  通知 {申请人}
```

### 规则2：请假审批

**配置**：
```
触发时机：当提交请假申请
执行动作：
  1. 判断请假类型
  2. 通知对应审批人
  3. 同步到 {考勤表}
请假类型规则：
  1-3天 → {直属上级}审批
  4-7天 → {部门经理}审批
  >7天 → {HR}审批
```

### 规则3：合同审批

**配置**：
```
触发时机：当提交合同审批
执行动作：
  1. 法务初审
  2. 财务复核
  3. 总经理审批
并行通知：
  - 法务收到通知时，同时通知财务准备
  - 两方都通过后，自动流转到总经理
```

## 定时任务配置

### 定时规则1：每日站会汇总

**配置**：
```
触发时机：每个工作日 09:00
执行动作：
  1. 汇总昨日完成任务
  2. 统计今日计划任务
  3. 识别风险任务
  4. 生成站会报告
发送渠道：项目群
```

**输出模板**：
```
【{日期} 项目日报】

✅ 昨日完成：{完成任务数} 项
🔄 今日计划：{计划任务数} 项
⚠️ 风险事项：{风险任务数} 项
💡 AI建议：{AI生成建议}
```

### 定时规则2：周报生成

**配置**：
```
触发时机：每周五 18:00
执行动作：
  1. 汇总本周数据
  2. 计算周环比
  3. 生成周报
发送渠道：管理群
```

### 定时规则3：月度数据归档

**配置**：
```
触发时机：每月最后一天 23:00
执行动作：
  1. 归档当月记录到 {历史表}
  2. 生成月度统计
  3. 创建下月空白记录
```

### 定时规则4：AI批量处理

**配置**：
```
触发时机：每天 09:00
前置条件：待处理记录数 ≥ 10
执行动作：
  1. 合并待处理记录
  2. 调用AI批量分析
  3. 拆分结果并更新
  4. 记录消耗点数
```

## 规则效果评估

### 评估指标

| 指标 | 计算方式 | 目标值 |
|-----|---------|-------|
| 触发成功率 | 成功执行/总触发 | > 99% |
| 平均执行时间 | 总执行时间/执行次数 | < 5秒 |
| 错误率 | 错误次数/总次数 | < 1% |
| 覆盖任务数 | 规则影响的记录数 | 持续增加 |

### 优化建议

```
效果不佳规则排查：
1. 触发条件是否过于宽泛？
2. 执行动作是否过于复杂？
3. 是否存在循环依赖？
4. 条件判断是否有歧义？
5. 是否需要增加容错处理？
```
