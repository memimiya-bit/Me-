## 流程概述

### 商品管理全景

```
商品规划 → 采购执行 → 库存管理 → 生命周期管理
    ↓          ↓          ↓          ↓
 商品企划会  采购决策会  周度库存会  退市决策会
```

### 服装企业商品管理特点

- SKU 众多：颜色、尺码组合多
- 季节性强：过季商品需要及时处理
- 库存敏感：积压占用资金，缺货损失销售
- 成本波动：面料价格影响采购成本
- 渠道复杂：线上线下多渠道管理

## 流程节点映射

### 节点一：商品规划

**触发时机**：商品企划会
**关键动作**：
- 确定商品结构
- 制定 SKU 计划
- 设定价格带

**输出**：商品规划书 + SKU 清单

### 节点二：采购执行

**触发时机**：SKU 清单确认
**关键动作**：
- 供应商询价
- 价格谈判
- 合同签订

**输出**：采购订单

### 节点三：库存管理

**触发时机**：商品入库
**关键动作**：
- 入库登记
- 库存监控
- 补货决策

**输出**：库存报表

### 节点四：生命周期管理

**触发时机**：销售数据分析
**关键动作**：
- 周转分析
- 促销决策
- 退市处理

**输出**：退市计划

## 字段配置

### 商品管理必填字段

| 字段名称 | 类型 | 验证规则 | 说明 |
|---------|------|---------|------|
| SKU编码 | 文本 | 唯一 | 自动生成：SKU-品类-序号 |
| 商品名称 | 文本 | 必填 | 商品名称 |
| 品类 | 单选 | - | 上装/下装/套装/配饰 |
| 季节 | 单选 | - | SS24/FW24 等 |
| 品牌 | 文本 | - | 商品品牌 |
| 颜色 | 多选 | - | 商品颜色 |
| 尺码 | 多选 | - | S/M/L/XL 等 |
| 吊牌价 | 数字 | 正数 | 单位：元 |
| 采购成本 | 数字 | 正数 | 单位：元 |
| 最低售价 | 数字 | 正数 | 促销最低价 |
| 供应商标识 | 关联 | - | 关联供应商表 |
| 采购订单号 | 文本 | - | 对应采购订单 |
| 采购数量 | 数字 | 正整数 | 采购件数 |
| 已到货数量 | 数字 | 正整数 | 入库数量 |
| 在途数量 | 数字 | 正整数 | 运输中数量 |
| 库存数量 | 数字 | 正整数 | 当前库存 |
| 安全库存 | 数字 | 正整数 | 最低库存 |
| 预留数量 | 数字 | 正整数 | 已预订数量 |
| 可售数量 | 数字 | 自动计算 | 库存-预留 |
| 周转天数 | 数字 | 自动计算 | 平均库存天数 |
| 生命周期阶段 | 单选 | - | 见下方选项 |
| 上市日期 | 日期 | - | 实际上架日期 |
| 退市日期 | 日期 | - | 计划下架日期 |
| 促销次数 | 数字 | - | 已参与促销次数 |
| 残值率 | 百分比 | 0-100% | 预估残值 |
| 商品状态 | 单选 | - | 正常/缺货/滞销/退市 |

### 生命周期阶段选项

| 阶段 | 名称 | 说明 | 典型时长 |
|-----|------|------|---------|
| 1 | 新品导入 | 刚上市 | 0-4周 |
| 2 | 成长期 | 销售上升 | 4-8周 |
| 3 | 成熟期 | 稳定销售 | 8-16周 |
| 4 | 衰退期 | 销售下降 | 16-24周 |
| 5 | 促销期 | 清仓处理 | 促销期间 |
| 6 | 退市 | 已下架 | - |

### 自动计算字段

```公式
可售数量 = 库存数量 - 预留数量
周转天数 = (平均库存 * 365) / 年销量
毛利 = 吊牌价 - 采购成本
毛利率 = (毛利 / 吊牌价) * 100%
库龄 = TODAY() - 上市日期
```

### 风险预警规则

```公式
缺货风险 = IF(可售数量 < 安全库存, "需要补货", "正常")
滞销风险 = IF(库龄 > 90 AND 生命周期阶段 NOT IN ("促销期","退市"), "滞销预警", "正常")
临期风险 = IF(退市日期 - TODAY() < 30 AND 商品状态="正常", "临期预警", "正常")
超储风险 = IF(库存数量 > 采购数量 * 1.5, "超储预警", "正常")
```

## 工作流配置

### 工作流一：库存预警

```
触发：每天检查
条件：可售数量 < 安全库存
动作：
  1. 发送补货提醒给商品专员
  2. 标记为"缺货预警"
  3. 自动计算建议补货量
```

### 工作流二：滞销提醒

```
触发：每周一检查
条件：库龄 > 60 AND 销售数量 < 10
动作：
  1. 标记为"滞销商品"
  2. 建议参与促销活动
  3. 通知营销部门
```

### 工作流三：临期退市

```
触发：退市日期前30天
条件：商品状态="正常" AND 库存数量 > 0
动作：
  1. 发送退市提醒
  2. 要求制定促销计划
  3. 如果退市日期到达：
     - 自动更新商品状态为"退市"
```

### 工作流四：补货自动计算

```
触发：当销售数据更新时
动作：
  1. 计算过去7天日均销量
  2. 计算预计可售天数
  3. 如果可售天数 < 14：
     - 生成补货建议
     - 通知采购部门
```

## 采购决策参考

### 新品采购决策

| 条件 | 建议 |
|-----|------|
| 毛利率 > 50% 且 周转天数 < 60 | 正常采购 |
| 毛利率 > 50% 且 周转天数 60-90 | 谨慎采购 |
| 毛利率 < 50% 或 周转天数 > 90 | 不建议采购 |

### 促销决策参考

| 库龄 | 建议折扣 | 处理方式 |
|-----|---------|---------|
| 60-90天 | 9折 | 线上优先 |
| 90-120天 | 8折 | 会员专享 |
| 120-180天 | 7折 | 全渠道促销 |
| > 180天 | 5折以下 | 清仓处理 |
