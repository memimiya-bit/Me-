## 核心理念

**痛点**：当前AI生成报告存在"机器腔"、术语堆砌、缺乏叙事逻辑，高管平均阅读时间仅47秒。

**目标**：让AI生成的报告成为高管真正愿意阅读的"商业叙事"。

---

## 可读性指数体系

### 可读性指数计算

```formula
可读性指数 = (
  {商业术语密度} × 0.2 + 
  {故事连贯性} × 0.3 + 
  {行动明确度} × 0.4 + 
  {情绪共鸣度} × 0.1
) × 100
```

### 各维度定义

| 维度 | 权重 | 定义 | 评估标准 |
|-----|-----|------|---------|
| 商业术语密度 | 20% | 术语与业务目标的相关度 | 1-10分，10为最佳 |
| 故事连贯性 | 30% | 数据点之间的逻辑关联 | 1-10分，10为最佳 |
| 行动明确度 | 40% | 是否给出清晰可执行的建议 | 1-10分，10为最佳 |
| 情绪共鸣度 | 10% | 与读者利益的关联程度 | 1-10分，10为最佳 |

### 可读性等级

| 可读性指数 | 等级 | 状态 |
|-----------|-----|------|
| >=85 | A | 优秀，可直接使用 |
| 70-84 | B | 良好，微调后使用 |
| 50-69 | C | 一般，需要重构 |
| <50 | D | 较差，需要重写 |

## 智能降噪机制

### 核心公式

```formula
最终报告 = AI_REWRITE(
  {原始数据}, 
  "用{收件人角色}能听懂的语言，聚焦{核心业务目标}，避免专业术语，用具体案例说明"
)
```

### 降噪规则库

| 原始表达 | 降噪后表达 |
|---------|-----------|
| "CPI=0.85, SPI=0.92" | "成本超支15%，进度滞后8%" |
| "资源利用率82%" | "团队产能利用充分，略有提升空间" |
| "缺陷密度2.3/千行" | "代码质量良好，上线后问题风险可控" |
| "EAC预测偏差12%" | "预计最终花费将超出预算约10万" |

### 智能改写示例

**原始数据**：
```
- 当前进度：EV=80万，PV=85万，AC=94万
- CPI=0.85, SPI=0.94
- 预测EAC=110万
- 风险：资源冲突指数128%
```

**财务总监版**：
> Q2项目成本超支15%（预算85万，实际94万），进度略有滞后（偏差8%）。按当前趋势，预计最终花费将达110万，超出预算25万。建议优先控制外包成本（占总支出40%），可将超支控制在15万以内。

**技术总监版**：
> 资源调度存在瓶颈：当前资源冲突指数128%，主要冲突集中在测试环境。建议实施测试环境自动回收策略（预计提升利用率25%），同时优化并行测试用例安排。

## 角色适配策略

### 角色语言映射表

| 角色 | 关注焦点 | 偏好语言 | 禁忌术语 |
|-----|---------|---------|---------|
| CEO | 战略/增长 | 机会/风险/ROI | 技术细节 |
| CFO | 成本/利润 | 预算/收益/偏差 | 架构/代码 |
| COO | 效率/流程 | 吞吐量/瓶颈/优化 | 财务指标 |
| CTO | 技术/架构 | 性能/扩展性/技术债 | 商业术语 |
| PMO | 进度/资源 | 里程碑/风险项/依赖 | 过于技术化 |

### 各角色报告模板

**CEO版**：
```
【{项目名称}月度报告】

核心结论：{一句话总结项目状态}

关键数字：
- {最重要的业务指标}：{数值}（{同比/环比变化}）
- 预计{财务影响}：{金额}万
- 预计完成时间：{日期}

风险与机会：
{最大的1-2个风险/机会，用业务语言描述}

行动建议：
1. {最需要决策的事项}
2. {最需要资源支持的事项}
```

**CFO版**：
```
【{项目名称}成本分析报告】

成本状态：{正常/预警/超支}（偏差{百分比}）

明细：
- 预算总额：{BAC}万
- 实际支出：{AC}万
- 预计完工：{EAC}万
- 完工偏差：{VAC}万

预测假设：
{EAC计算的关键假设}

风险点：
{可能导致进一步超支的因素}

建议措施：
{具体的成本控制建议，含预期效果}
```

**技术总监版**：
```
【{项目名称}技术状态报告】

系统健康度：{评分}/100

性能指标：
- 可用性：{百分比}
- 响应时间：{P99值}ms
- 错误率：{百分比}%

技术债：{金额}万（{时间}人天）

风险项：
{技术风险，用技术语言描述}

优化建议：
{具体的优化方案，含预期效果}
```

## 实施模板

### 报告生成工作流

```
1. 数据收集 → 2. 原始分析 → 3. 角色识别 → 4. 智能改写 → 5. 可读性验证 → 6. 输出报告
```

### AI改写指令模板

```
你是一名资深商业报告撰写专家，请将以下数据转化为面向{目标角色}的报告：

【目标读者】
{角色名称}，关注{关注焦点}

【报告要求】
1. 使用{角色}能听懂的语言，避免专业术语
2. 聚焦{核心业务目标}
3. 用具体数字和案例说明，避免模糊表述
4. 给出明确可执行的建议
5. 突出与读者利益的关联

【原始数据】
{数据内容}

【输出格式】
核心结论（1句话）
关键数字（3-5个）
风险与机会（2-3条）
行动建议（1-3条）
```

### 可读性自检清单

```
□ 报告能否在30秒内传达核心信息？
□ 是否使用了读者熟悉的语言？
□ 数字是否有对比（同比/环比/预算）？
□ 建议是否具体可执行？
□ 是否说明了行动的价值/收益？
□ 是否避免了读者不关心的细节？
□ 排版是否清晰易读？
```

## 效果评估

### 评估指标

| 指标 | 测量方式 | 目标值 |
|-----|---------|-------|
| 阅读完成率 | 完成阅读的用户/打开用户 | >80% |
| 决策通过率 | 报告中建议被采纳的比例 | >75% |
| 阅读时长 | 用户平均阅读时长 | 2-5分钟 |
| 理解准确率 | 用户理解与实际数据的偏差 | <10% |

### 持续优化机制

```formula
优化建议 = AI_ANALYZE(
  {用户反馈} + {阅读行为数据},
  "提取报告改进建议，按影响力排序"
)
```
