## 流程概述

### 研发流程全景

```
需求定义 → 方案设计 → 开发验证 → 量产准备
    ↓          ↓          ↓          ↓
 季度企划会  设计评审   样衣确认   产前会议
```

### 服装企业研发特点

- 季节性强：春夏/秋冬两季需要提前规划
- 品类复杂：上装、下装、套装、配饰等
- 面料关键：面料选择影响成本和交期
- 工艺要求：不同工艺难度差异大
- 成本敏感：需要在设计阶段控制成本

## 流程节点映射

### 节点一：需求定义

**触发时机**：季度企划会
**关键动作**：
- 确定季节主题
- 规划品类结构
- 制定开发计划

**输出**：季度开发计划书

### 节点二：方案设计

**触发时机**：需求确认
**关键动作**：
- 设计稿制作
- 面料初选
- 成本估算

**输出**：设计方案 + 初版成本

### 节点三：开发验证

**触发时机**：方案评审通过
**关键动作**：
- 样衣制作
- 面料确认
- 工艺确认

**输出**：确认样 + 工艺单

### 节点四：量产准备

**触发时机**：样衣确认
**关键动作**：
- 产前评审
- 物料准备
- 产能对接

**输出**：量产确认书

### 设计状态选项

| 阶段 | 名称 | 完成度 | 说明 |
|-----|------|-------|------|
| 1 | 规划中 | 10% | 列入开发计划 |
| 2 | 设计中 | 30% | 设计稿制作中 |
| 3 | 评审中 | 50% | 设计评审中 |
| 4 | 打样中 | 70% | 样衣制作中 |
| 5 | 确认中 | 85% | 样衣确认中 |
| 6 | 已确认 | 95% | 设计定稿 |
| 7 | 已量产 | 100% | 进入量产 |
| 8 | 已上市 | 100% | 产品已上市 |

### 风险预警规则

```公式
面料风险 = IF(采购状态="待采购" AND 预计量产日期-TODAY()<30, "高风险", "正常")
工艺风险 = IF(工艺难度="复杂" AND 工艺单状态≠"已确认", "高风险", "正常")
交期风险 = IF(样衣状态≠"已确认" AND 预计量产日期-TODAY()<45, "高风险", "正常")
综合风险 = IF(面料风险="高风险" OR 工艺风险="高风险" OR 交期风险="高风险", "需要关注", "正常")
```

## 工作流配置

### 工作流一：状态自动推进

```
触发：当完成度变更时
条件：满足阶段切换条件
动作：
  1. 自动更新设计状态
  2. 记录变更历史
  3. 如果状态=打样中：
     - 通知样衣工坊
  4. 如果状态=已确认：
     - 通知采购部门准备物料
     - 通知生产部门准备产能
```

### 工作流二：交期预警

```
触发：每周一检查
条件：预计量产日期-TODAY()<60 AND 设计状态<6
动作：
  1. 发送预警给设计师和项目经理
  2. 标记为"交期预警"
  3. 如果预警超过2周：
     - 升级通知给研发总监
```

### 工作流三：成本超支提醒

```
触发：当成本预算确定后
条件：后续录入的成本>成本预算*1.1
动作：
  1. 发送通知给设计师和成本部门
  2. 要求重新评估
  3. 记录超支原因
```

### 工作流四：量产提醒

```
触发：预计量产日期前14天
条件：设计状态="已确认" AND 采购状态≠"已到料"
动作：
  1. 发送提醒给采购部门
  2. 提醒及时跟进物料
```
