## 表结构设计

### 「意图映射表」表结构

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 标准意图 | 单行文本 | 统一的动作名称 |
| 意图分类 | 单选 | 所属大类 |
| 同义词/变体 | 多行文本 | 所有表达方式 |
| 触发动作 | 多行文本 | 关联的操作模板 |
| 优先级 | 数字 | 处理优先级1-10 |
| 状态 | 单选 | 启用/禁用/测试中 |
| 创建时间 | 日期 | 自动记录 |
| 更新频率 | 数字 | 月均触发次数 |

## 维护机制

### 动态更新规则

| 更新来源 | 更新频率 | 操作 |
|---------|---------|------|
| 规则挖矿发现 | 每周 | 新增同义词变体 |
| 用户修正记录 | 实时 | 补充遗漏表达 |
| AI兜底分析 | 每月 | 合并相似意图 |
| 业务变化 | 按需 | 删除过时变体 |

### 同义词质量评估

```formula
// 同义词有效性评分
有效性 = (
  触发次数 × 0.5 +
  准确率 × 0.3 +
  覆盖率 × 0.2
)

// 清理条件
IF(
  月触发次数 < 2 AND 准确率 < 60%,
  标记为"待清理",
  "继续使用"
)
```

### 定期维护清单

```
每月检查：
□ 清理低效同义词（触发<2次且准确率<60%）
□ 合并重复意图（相似度>90%）
□ 拆分复杂意图（触发次数>100且准确率<80%）
□ 补充新变体（从复核池学习）
□ 更新优先级（根据业务重要性调整）
```

### 意图冲突处理

当同一消息可能匹配多个意图时：

```formula
// 意图选择优先级
最终意图 = IF(
  匹配数量 = 1, 选择该意图,
  匹配数量 > 1, 选择优先级最高的意图,
  推送到复核池
)
```

## 与硬规则的配合

### 规则匹配流程

```
消息进入
    │
    ▼
匹配意图映射表
    │
    ├── 命中1个 → 执行对应规则
    │
    ├── 命中多个 → 选择优先级最高的
    │
    └── 未命中 → 发送给AI兜底
```

### 配置示例

```formula
// 硬规则配置（飞书自动化）
触发器：收到群消息

动作：
1. 搜索「意图映射表」
   - 在同义词/变体中查找消息关键词
   
2. 匹配逻辑：
   - 消息包含任意一个变体 → 匹配成功
   
3. 获取标准意图：
   - 返回匹配行的「标准意图」
   
4. 执行动作：
   - 查找「操作模板库」中场景=标准意图
   - 执行对应操作步骤
```
