## 规模分类标准

### 规模维度定义

| 规模 | 员工数量 | 项目数量 | 年营收(万元) | 部门数量 | 适用版本 |
|-----|--------|---------|------------|---------|---------|
| S(小型) | <50 | <5 | <2000 | 3-5 | V2.1基础版 |
| M(中型) | 50-200 | 5-20 | 2000-20000 | 5-10 | V2.1标准版 |
| L(大型) | 200-1000 | 20-50 | 20000-200000 | 10-20 | V2.1企业版 |
| XL(超大型) | >1000 | >50 | >200000 | >20 | V2.1定制版 |

### 规模识别公式

```公式
企业规模 = SWITCH(TRUE(),
  AND(员工数<50, 项目数<5), "S(小型)",
  AND(员工数<=200, 项目数<=20), "M(中型)",
  AND(员工数<=1000, 项目数<=50), "L(大型)",
  "XL(超大型)"
)
```

### 规模特征对照

| 规模 | 管理特征 | 流程复杂度 | 技术要求 | 成本投入 |
|-----|---------|-----------|---------|---------|
| S | 扁平化 | 流程简单 | 基础工具 | 低 |
| M | 层级初现 | 流程规范 | 标准工具 | 中 |
| L | 矩阵管理 | 多流程协同 | 集成工具 | 高 |
| XL | 集团化 | 生态协同 | 自定义开发 | 极高 |

## 复杂度评分矩阵

### 七维度评估体系

| 维度 | 权重 | 评分标准 | 数据来源 |
|-----|-----|---------|---------|
| 跨部门协作 | 20% | 1-10分 | 协作部门数 |
| 技术难度 | 20% | 1-10分 | 技术复杂度评估 |
| 系统集成 | 15% | 1-10分 | 集成系统数 |
| 项目类型 | 15% | 1-10分 | 业务类型数 |
| 地域覆盖 | 10% | 1-10分 | 分布区域数 |
| 监管要求 | 10% | 高=10/中=5/低=1 | 行业属性 |
| 客户定制 | 10% | 1-10分 | 定制比例 |

### 复杂度评分计算

```公式
复杂度评分 = (
  跨部门协作得分 × 0.2 +
  技术难度得分 × 0.2 +
  系统集成得分 × 0.15 +
  项目类型得分 × 0.15 +
  地域覆盖得分 × 0.1 +
  监管要求得分 × 0.1 +
  客户定制得分 × 0.1
)
```

### 各维度评分细则

**跨部门协作**：
```公式
跨部门协作得分 = SWITCH(TRUE(),
  协作部门数 >= 8, 10,
  协作部门数 >= 5, 8,
  协作部门数 >= 3, 5,
  协作部门数 >= 1, 3,
  1
)
```

**技术难度**：
```公式
技术难度得分 = 
  IF(技术栈数量 > 5, 2, 0) +
  IF(新技术采用 = "是", 2, 0) +
  IF(性能要求 = "高", 2, 0) +
  IF(安全要求 = "高", 2, 0) +
  IF(集成复杂度 = "高", 2, 0)
```

**系统集成**：
```公式
系统集成得分 = SWITCH(TRUE(),
  集成系统数 > 5, 10,
  集成系统数 >= 3, 7,
  集成系统数 >= 1, 4,
  0
)
```

## 复杂度分级体系

### 复杂度等级定义

| 等级 | 评分范围 | 名称 | 特征描述 |
|-----|---------|-----|---------|
| C1 | 1-3 | ★☆☆ 基础 | 单一业务，标准化流程 |
| C2 | 4-6 | ★★☆ 中级 | 多业务，部分定制化 |
| C3 | 7-10 | ★★★ 高级 | 跨业务，高度定制化 |

### 复杂度分级公式

```formula
复杂度等级 = SWITCH(TRUE(),
  复杂度评分 <= 3, "★☆☆(基础)",
  复杂度评分 <= 7, "★★☆(中级)",
  "★★★(高级)"
)
```

### 不同规模复杂度分布

| 规模 | ★☆☆基础占比 | ★★☆中级占比 | ★★★高级占比 |
|-----|-------------|-------------|-------------|
| S | 70% | 25% | 5% |
| M | 40% | 45% | 15% |
| L | 20% | 50% | 30% |
| XL | 10% | 40% | 50% |

### 复杂度适配策略

**基础级(C1)**：

**中级(C2)**：
- 推荐公式集：标准公式集（含EVM）
- AI使用频率：3-5次/周
- 自动化规则：15-25条
- AI点数预算：30-50点/月

**高级(C3)**：
- 推荐公式集：高级公式集（含预测）
- AI使用频率：5-10次/周
- 自动化规则：30-50条
- AI点数预算：80-120点/月

## 升级路径指南

### 规模升级检查清单

**S → M 升级条件**：
```
□ 员工数超过50人
□ 项目数超过5个
□ 需要跨部门协作流程
□ 需要EVM成本分析
□ 需要资源冲突检测
□ 需要扩展权限层级
```

**M → L 升级条件**：
```
□ 员工数超过200人
□ 项目数超过20个
□ 引入PMO组织
□ 需要Portfolio分析
□ 需要多系统集成
□ 需要质量流程标准化
```

**L → XL 升级条件**：
```
□ 员工数超过1000人
□ 项目数超过50个
□ 集团化管理体系
□ 需要组合项目管理
□ 需要AI智能调度
□ 需要ISO/CMMI认证
```

### 复杂度升级检查清单

**C1 → C2 升级条件**：
```
□ 业务类型增加到2种以上
□ 开始有跨部门协作需求
□ 客户定制需求增加
□ 系统集成需求出现
□ 出现多地域协作
```

**C2 → C3 升级条件**：
```
□ 业务类型增加到3种以上
□ 跨部门协作成为常态
□ 技术复杂度显著增加
□ 监管要求提高
□ 高度客户定制化需求
```

### 升级执行清单

**配置升级步骤**：
```
1. 评估当前规模和复杂度
2. 确定目标规模和版本
3. 备份现有配置
4. 准备数据迁移方案
5. 配置新功能模块
6. 测试新配置
7. 用户培训
8. 切换上线
9. 验证效果
10. 持续优化
```
