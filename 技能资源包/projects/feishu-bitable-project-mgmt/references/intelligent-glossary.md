## 五维词库结构

### 核心字段设计

| 字段 | 说明 | 示例 | 数据来源 |
|-----|------|------|---------|
| **核心术语** | 标准词汇及变体 | CRM/客户管理系统/客管系统 | 行业标准+业务 |
| **业务权重** | 场景优先级(1-10) | 合同金额(销售=9,法务=8) | 历史决策 |
| **上下文规则** | 触发条件组合 | 合同 + (金额>100万 OR 新供应商) | 专家+学习 |
| **时效属性** | 有效时间窗口 | 促销政策(2025-06-01至2025-06-15) | 业务+标注 |
| **关联知识** | 关联流程/文档 | 合同审核→法务流程SOP | 知识图谱 |

### 数据结构示例

```yaml
术语: 合同金额异常

五维数据:
  核心术语:
    标准词: "合同金额异常"
    变体: ["金额超限", "合同超支", "预算偏差"]
    缩写: ["金异", "合超"]
    
  业务权重:
    销售场景: 9
    法务场景: 8
    财务场景: 10
    技术场景: 4
    
  上下文规则:
    触发条件: >
      合同 + 金额偏差 > 10% 
      OR 金额 > 100万
      OR 新供应商合同
    排除条件: >
      合同法学习
      合同模板讨论
      
  时效属性:
    有效期: "2025-01-01至2026-12-31"
    更新频率: "季度审核"
    状态: "活跃"
    
  关联知识:
    关联流程: ["合同审核流程", "预算审批流程"]
    关联文档: ["合同管理制度", "预算管控SOP"]
    关联角色: ["法务", "财务BP", "部门负责人"]
```

## 多源采集机制

### 数据源矩阵

```yaml
数据源配置:
  业务系统:
    来源: "ERP/CRM/OA审批流"
    采集逻辑: "提取高频字段组合 + 关联上下文"
    示例: "付款申请单中'紧急程度=高'且'金额>50万'触发特殊流程"
    权重: 0.4
    
  协作痕迹:
    来源: "飞书文档/群聊/会议记录"
    采集逻辑: "识别被高频@的术语 + 人工修正记录"
    示例: "群聊中'客资'被修正为'客户资料'10次，自动学习映射"
    权重: 0.3
    
  行业标准:
    来源: "国家标准/行业白皮书/API文档"
    采集逻辑: "定期同步 + 语义对齐"
    示例: "对接《金融行业术语标准》更新37个新术语"
    权重: 0.2
    
  外部信号:
    来源: "新闻/政策/竞品动态"
    采集逻辑: "关键词监控 + 影响度评估"
    示例: "监控'数据安全法'相关讨论，自动提升合规术语权重"
    权重: 0.1
```

### 智能采集引擎

```formula
// 候选术语识别
候选术语 = FILTER(
  业务日志,
  AND(
    出现频次 > 5,
    NOT(术语库已存在),
    词长度 BETWEEN(2, 8),
    NOT(通用词匹配)
  )
)

// 术语新鲜度评估
术语新鲜度 = (
  {使用频率变化率} * 0.4 + 
  {外部关注度} * 0.3 + 
  {业务影响度} * 0.2 + 
  {专家评分} * 0.1
) * 100

// 采集优先级
采集优先级 = 
  业务系统 * 0.4 + 
  协作痕迹 * 0.3 + 
  行业标准 * 0.2 + 
  外部信号 * 0.1
```

## 自愈机制

### 词库健康监控

```formula
// 健康度计算
词库健康度 = (
  {术语准确率} * 0.4 + 
  {规则完整率} * 0.3 + 
  {更新及时率} * 0.2 + 
  {用户满意度} * 0.1
) * 100

// 自愈触发条件
自愈触发 = OR(
  误判率 > 30%,
  术语新鲜度 < 20%,
  用户投诉 > 5次/周
)
```

### 自愈方案生成

```formula
// 诊断分析
诊断结果 = AI_ANALYZE(
  "分析词库异常原因：
   高误判术语TOP10
   误判模式分类（上下文/权重/时效）
   潜在影响范围
   修复优先级排序"
)

// 修复方案
修复方案 = AI_GENERATE_FIX_PLAN(
  "分析'{term}'误判模式，建议：
   
   方案1：调整权重
   - 当前权重：{current_weight}
   - 建议权重：{new_weight}
   - 预期效果：误判率降低{expected_improvement}%
   
   方案2：补充上下文规则
   - 负面示例：{negative_example}
   - 补充规则：{new_rule}
   
   方案3：拆分术语
   - 原术语：'接口'
   - 拆分为：'API接口' + '客户接口'
   - 适用场景：技术/销售
   
   方案4：暂时禁用
   - 禁用原因：规则尚未完善
   - 预计修复时间：{estimated_time}
   - 替代方案：{alternative}"
)
```

## 版本管理

### 版本策略

| 策略 | 适用场景 | 生效方式 | 回滚时间 |
|-----|---------|---------|---------|
| **热更新** | 权重调整/简单规则修改 | 实时生效 | 即时 |
| **灰度发布** | 新增术语/复杂规则 | 10%用户测试 | 5分钟 |
| **全量发布** | 重大架构调整 | 审批后全量 | 10分钟 |
| **自动回滚** | 误判率>15% | 自动触发 | 即时 |

### 版本控制

```formula
// 版本号格式
版本号 = 主版本.次版本.修订版本-环境
示例: V2.1.3-prod, V2.2.0-preview

// 影响范围预评估
影响评估 = AI_IMPACT_ASSESSMENT(
  "评估词库变更对{业务场景}的影响：
   覆盖消息量变化
   误判率预测变化
   人工复核量变化
   业务中断风险
   
  输出：
   - 风险等级：低/中/高
   - 影响范围：{具体数据}
   - 缓解建议：{具体措施}"
)
```

## 实施配置

### 飞书多维表格字段

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 核心术语 | 单行文本 | 标准词汇 |
| 变体列表 | 多行文本 | 缩写/错别字 |
| 业务权重_销售 | 数字 | 1-10 |
| 业务权重_法务 | 数字 | 1-10 |
| 上下文规则 | 多行文本 | 触发条件 |
| 时效有效期 | 日期范围 | 有效时间 |
| 关联知识 | 多行文本 | 关联文档 |
| 健康状态 | 单选 | 健康/学习中/待优化 |
| 版本号 | 单行文本 | 当前版本 |

### 自动化规则

```formula
// 自动健康检测
健康状态 = IF(
  误判率 > 30%, "待优化",
  误判率 > 10%, "学习中",
  "健康"
)

// 自动版本更新
IF(
  权重调整, 热更新(实时生效)
  新增术语, 灰度发布(10%用户)
  架构调整, 全量发布(审批后)
)

// 自动回滚
IF(
  新版本误判率 > 15%,
  自动回滚(上一稳定版本)
)
```

| 权重准确率 | >90% | 用户确认准确数/总调整 |

| 自愈成功率 | >85% | 自愈有效数/触发数 |
