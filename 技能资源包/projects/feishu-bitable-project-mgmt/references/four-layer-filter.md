## L1语法层

### 功能说明

| 检查类型 | 说明 | 处理方式 |
|---------|------|---------|
| 拼写检查 | 错别字识别 | 标记+建议修正 |
| 标点规范 | 标点使用规范 | 自动修正 |
| 格式统一 | 大小写/空格/特殊字符 | 标准化处理 |
| 编码检测 | 表情/特殊符号 | 转为文字描述 |

### 配置示例

```formula
// 语法层检查
语法检查结果 = {
  "拼写错误": FALSE,
  "标点规范": TRUE,
  "格式统一": TRUE,
  "编码检测": TRUE
}

// 语法修正
修正后内容 = TRIM(UPPER(原始内容))
```

## L2规则层

### 功能说明

基于业务上下文的规则引擎，与V5.3方案保持一致：

| 场景 | 触发规则 | 示例 |
|-----|---------|------|
| 法务 | 合同 + 审核/签署/条款 | 合规 |
| 销售 | 合同 + 金额/客户/回款 | 合规 |
| 技术 | 接口 + 调用/对接/数据 | 合规 |
| 运营 | 活动 + 参与/转化/ROI | 合规 |

### 配置示例

```formula
// 规则层匹配
规则匹配 = SWITCH(
  场景,
  "法务", AND(关键词="合同", 关联词 IN {"审核","签署","条款"}),
  "销售", AND(关键词="合同", 关联词 IN {"金额","客户","回款"}),
  "技术", AND(关键词="接口", 关联词 IN {"调用","对接","数据"}),
  FALSE
)
```

## L3语义层

### 功能说明

跨模态语义理解，不仅分析文本，还结合表格和图片：

```prompt
语义分析任务：

分析消息在{业务场景}中的真实意图，区分：

1. 工作指令（需处理）
   - 明确的行动要求
   - 指定负责人和截止时间
   
2. 信息分享（需记录）
   - 知识传递
   - 进度通报
   
3. 闲聊讨论（可过滤）
   - 非业务相关
   - 社交互动
   
4. 知识查询（需回答）
   - 历史案例查询
   - 标准流程咨询

输出格式：
- 意图类型：工作指令/信息分享/闲聊讨论/知识查询
- 置信度：0-100
- 建议动作：{具体建议}
- 关键信息：{提取的关键数据}
```

### 语义理解维度

| 维度 | 说明 | 权重 |
|-----|------|------|
| 业务相关性 | 与当前业务的关联程度 | 0.3 |
| 行动必要性 | 是否需要采取行动 | 0.3 |
| 信息价值 | 对决策的影响程度 | 0.2 |
| 时效要求 | 时间敏感度 | 0.2 |

## L4意图层

### 功能说明

业务意图识别，预测用户的真实行动需求：

```yaml
意图类型:
  合同审核意图:
    检测条件: "合同金额>100万 OR 新供应商 OR 特殊条款"
    触发动作: "通知法务介入"
    
  风险预警意图:
    检测条件: "逾期/违约/诉讼关键词"
    触发动作: "生成风险报告"
    
  决策请求意图:
    检测条件: "金额超限 OR 权限外 OR 影响范围大"
    触发动作: "提交审批流程"
    
  知识获取意图:
    检测条件: "查询类词汇"
    触发动作: "检索知识库"
```

### 意图识别提示词

```prompt
业务意图识别任务：

分析用户消息的真实意图：

1. 合同审核意图
   检测：是否需要法务介入（合同金额>100万/新供应商/特殊条款）
   
2. 风险预警意图
   检测：识别潜在风险信号（逾期/违约/诉讼）
   
3. 决策请求意图
   检测：判断是否需要上级审批（金额/权限/影响范围）
   
4. 知识获取意图
   检测：区分是否在查询历史案例或标准流程

消息内容：{text}
当前项目：{project}
用户角色：{role}

输出格式：
- 主要意图：{意图类型}
- 置信度：0-100
- 次要意图：{可能的辅助意图}
- 建议动作：{下一步建议}
```

## 实施配置

### 飞书多维表格字段

| 字段名 | 类型 | 说明 |
|-------|------|------|
| L1语法检查 | 单选 | 通过/需修正 |
| L2规则匹配 | 单选 | 匹配/未匹配 |
| L3语义置信度 | 数字 | 0-100 |
| L4意图类型 | 单选 | 工作指令/信息分享/闲聊/查询 |
| 最终决策 | 单选 | 处理/记录/过滤/回答 |

### 自动化规则

```formula
// L1→L2→L3→L4串联
处理结果 = 
  IF(L1通过,
    L2规则匹配,
    L1修正后继续
  ) → 
  IF(匹配,
    L3语义验证,
    L2未匹配标记
  ) →
  L4意图识别 →
  最终决策

// 意图层决策
最终决策 = SWITCH(
  L4意图,
  "工作指令", "立即处理",
  "信息分享", "记录归档",
  "知识查询", "检索回答",
  "闲聊讨论", "过滤忽略"
)
```

### 效果评估

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| L1语法准确率 | 100% | 拼写/格式检查 |
| L2规则召回率 | >95% | 业务关键词 |
| L3语义准确率 | >90% | 意图识别 |
| L4意图准确率 | >85% | 行动预测 |
| 端到端误判率 | <3% | 全链路 |
