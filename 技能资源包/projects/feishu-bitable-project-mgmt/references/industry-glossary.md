## 词库三要素

### 核心数据结构

专业词库必须包含以下三要素，仅收集词汇本身无效：

| 字段 | 说明 | 示例 |
|-----|------|------|
| **术语** | 核心词汇及变体（缩写/错别字） | CRM/客户关系管理 |
| **业务权重** | 在当前场景的优先级（1-10分） | 回款（销售=9分） |
| **上下文规则** | 触发条件（需关联动词/字段） | 回款+金额/逾期 |

### 术语分类示例

**销售领域术语库**：

| 术语 | 变体 | 业务权重 | 上下文规则 |
|-----|------|---------|-----------|
| 客户 | 客顾/甲方 | 8 | 客户+跟进/维护/开发 |
| 订单 | 工单/下单 | 9 | 订单+金额/数量/交期 |
| 回款 | 收款/收款 | 10 | 回款+逾期/账期/金额 |
| 商机 | 机会/线索 | 7 | 商机+转化/跟进/评估 |
| 报价 | 报价单/方案价 | 6 | 报价+确认/审批/调整 |

**技术领域术语库**：

| 术语 | 变体 | 业务权重 | 上下文规则 |
|-----|------|---------|-----------|
| API | 接口/服务接口 | 9 | API+调用/开发/文档 |
| 部署 | 上线/发布 | 8 | 部署+环境/版本/回滚 |
| Bug | 缺陷/问题 | 7 | Bug+修复/复现/严重度 |
| 迭代 | Sprint/版本 | 6 | 迭代+规划/评审/回顾 |

## 自动化收集机制

### 来源优先级

```
优先级1：业务系统日志
    ↓
优先级2：对话沉淀
    ↓
优先级3：行业标准库
```

### 1. 业务系统日志提取

**数据来源**：
- OA审批流中的高频术语（如"付款申请"关联"合同编号"）
- 工单系统中的业务词汇
- CRM系统中的自定义字段

**自动提取规则**：

```formula
// 候选术语识别
候选术语 = FILTER(
  业务日志,
  AND(
    出现频次 > 5,
    NOT(术语库已存在),
    词长度 BETWEEN(2, 8),
    NOT(通用词匹配)
  )
)

// 术语关联提取
关联字段 = AI_EXTRACT(
  "从{业务日志上下文}中提取该术语常搭配的动词和字段",
  候选术语
)
```

### 2. 对话沉淀机制

**自动抓取场景**：
- 用户@机器人后人工修正的语句
- 修正前后的关键词差异
- 高频出现的缩写/错别字

**处理流程**：

```formula
// 修正语句识别
原消息 = 用户原始输入
修正后 = 人工修正内容

// 关键词差异提取
差异词 = IF(
  原消息 ≠ 修正后,
  AI_EXTRACT("提取{原消息}与{修正后}的关键词差异"),
  NULL
)

// 自动标记
IF(差异词 ≠ NULL,
  写入候选词库,
  忽略
)
```

**示例**：
- 用户原消息："处理客资问题" → 修正为"处理客户资料问题"
- 自动识别："客资"为CRM领域缩写
- 自动写入：客资 → 客户资料，权重=6

### 3. 行业标准库对接

**推荐对接来源**：

| 来源 | 说明 | 获取方式 |
|-----|------|---------|
| 全国科技名词委 | 权威术语定义 | API对接/手动导入 |
| 国标术语库 | 国家标准术语 | 官方下载 |
| 行业协会术语 | 垂直领域标准 | 协会资料 |

**对接配置**：

```formula
// 标准术语导入
标准术语 = IMPORT("行业标准库URL")

// 匹配本地词库
本地匹配 = VLOOKUP(
  标准术语.术语,
  本地词库,
  2,
  FALSE
)

// 未匹配项
新增术语 = FILTER(
  标准术语,
  ISBLANK(本地匹配)
)
```

### 审核标准

| 审核维度 | 通过标准 | 拒绝标准 |
|---------|---------|---------|
| 专业性 | 行业通用术语 | 口语化表达 |
| 准确性 | 定义明确 | 歧义/多义 |
| 实用性 | 场景高频 | 低频/偶发 |
| 完整性 | 含变体和上下文 | 孤立词汇 |

## 飞书配置方案

### 词库数据结构

**多维表格字段设计**：

| 字段名 | 类型 | 说明 |
|-------|------|------|
| 术语 | 单行文本 | 核心词汇 |
| 变体 | 多行文本 | 缩写/错别字（换行分隔） |
| 所属领域 | 单选 | 销售/技术/运营/法务/行政 |
| 业务权重 | 数字 | 1-10分 |
| 上下文规则 | 多行文本 | 触发条件说明 |
| 关联动词 | 多选 | 审核/跟进/处理等 |
| 状态 | 单选 | 活跃/待审核/废弃 |
| 创建时间 | 日期 | 自动记录 |
| 更新人 | 人员 | 最后修改人 |

### 自动化规则

```formula
// 自动更新权重
权重调整 = SWITCH(
  月使用频次,
  >100, MIN(10, 基础权重 + 2),
  >50, 基础权重,
  >20, MAX(1, 基础权重 - 1),
  基础权重
)

// 低频术语预警
预警 = IF(
  AND(状态 = "活跃", 月使用频次 < 5),
  "建议评估是否废弃",
  "正常"
)

// 新术语自动待审
新术语状态 = IF(
  NOT(术语库已存在),
  "待审核",
  状态
)
```
