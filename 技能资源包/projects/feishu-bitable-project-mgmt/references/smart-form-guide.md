## 概述

### 设计原则

| 原则 | 说明 |
|-----|------|
| 自动化优先 | 核心字段自动填充，减少人工录入 |
| 数据质量 | 通过验证规则和触发机制保证准确性 |
| 可扩展性 | 支持业务变化和功能扩展 |
| 可追溯性 | 记录变更历史，支持审计回溯 |

### 字段分类总览

| 类别 | 字段数 | 说明 |
|-----|-------|------|
| 核心基础字段 | 6个 | 系统自动生成，记录基础元信息 |
| 智能分类字段 | 5个 | 基于规则自动分类和标签 |
| 智能识别字段 | 6个 | AI识别填充（需对接AI引擎） |
| 动态计算字段 | 6个 | 实时计算派生指标 |
| 关系映射字段 | 5个 | 跨表关联和引用 |
| 自动化触发字段 | 5个 | 自动化流程控制 |
| **合计** | **33个** | 完整的智能表单字段体系 |

## 字段详细说明

### 1. 核心基础字段

系统自动生成，无需人工维护。

| 字段名 | 类型 | 说明 | 自动化逻辑 |
|-------|------|------|-----------|
| record_id | 文本 | 记录唯一标识 | UUID自动生成 |
| create_time | 日期时间 | 记录创建时间 | NOW()自动填充 |
| update_time | 日期时间 | 最后修改时间 | ON UPDATE自动刷新 |
| creator_id | 文本 | 创建人ID | 从登录会话获取 |
| updater_id | 文本 | 更新人ID | 最后操作人 |
| record_status | 单选 | 记录状态 | 默认"正常" |

**状态选项**：1-正常 2-停用 3-删除

---

### 2. 智能分类字段

基于规则引擎自动分类。

| 字段名 | 类型 | 可选值 | 自动化逻辑 |
|-------|------|-------|-----------|
| contact_type | 单选 | internal/upstream/midstream/downstream/service | 关键词匹配 |
| auto_category | 多选 | AI识别标签 | NLP分析 |
| industry_tag | 多选 | 行业标签库 | 企业名称匹配 |
| priority_level | 数字 | 1-5 | 综合评分 |
| risk_score | 数字 | 0.0-10.0 | 合作状态评估 |

**优先级计算公式**：
```
优先级 = 紧急程度权重×0.4 + 对接频率权重×0.3 + 合作状态权重×0.3

紧急程度：P0=5, P1=4, P2=3, P3=2, P4=1
对接频率：daily=5, weekly=4, monthly=3, quarterly=2, yearly=1
合作状态：战略=5, 重点=4, 一般=3, 观察=2, 终止=1
```

---

### 3. 智能识别字段

AI引擎自动提取（需对接外部AI服务）。

| 字段名 | 类型 | 验证规则 | 置信度要求 |
|-------|------|---------|-----------|
| company_name_clean | 文本 | 标准企业名称格式 | >0.95 |
| contact_person_ai | 文本 | 中文姓名格式 | >0.85 |
| phone_ai | 文本 | 手机号正则 | >0.95 |
| email_ai | 文本 | 邮箱正则 | >0.95 |
| business_scope_ai | 多选 | 业务标签库 | >0.80 |

**置信度分级处理**：

| 置信度范围 | 处理策略 |
|-----------|---------|
| >=0.95 | 直接填充，标记"已确认" |
| 0.80-0.94 | 填充但标记"待审核" |
| 0.60-0.79 | 仅提示，人工确认 |
| <0.60 | 不填充，标记"识别失败" |

### 4. 动态计算字段

基于公式实时计算。

| 字段名 | 类型 | 计算公式 | 更新触发 |
|-------|------|---------|---------|
| next_contact_date | 日期 | 根据频率计算 | 频率变更时 |
| days_until_next | 数字 | DATEDIFF(next, TODAY) | 每日定时 |
| response_time_avg | 数字 | 历史均值 | 新记录后 |
| cooperation_duration | 数字 | (TODAY-首次)/30 | 每月1号 |
| contact_count_month | 数字 | 当月次数统计 | 每日定时 |
| sla_breach_count | 数字 | SLA超时累加 | 记录变更时 |

**公式配置示例**：
```
下次对接日期：
SWITCH(
  对接频率,
  "daily", DATEADD(最后对接日期, 1, "day"),
  "weekly", DATEADD(最后对接日期, 7, "day"),
  "monthly", DATEADD(最后对接日期, 30, "day"),
  "quarterly", DATEADD(最后对接日期, 90, "day"),
  ""
)

距下次对接天数：
IF(ISEMPTY(下次对接日期), "", DATETIME_DIFF(下次对接日期, TODAY(), "day"))
```

## 自动化工作流配置

### 工作流类型

| 工作流类型 | 触发条件 | 执行动作 |
|-----------|---------|---------|
| 创建工作流 | 记录创建 | 自动填充基础字段 |
| 更新工作流 | 字段变更 | 更新关联字段 |
| 定时工作流 | 每日定时 | 刷新计算字段 |
| AI识别工作流 | 新增原始数据 | 调用AI识别服务 |
| 提醒工作流 | 条件触发 | 发送通知 |

### 常用工作流配置

```
1. 记录创建工作流
触发：当记录创建时
动作：
- 自动生成 record_id
- 填充 create_time
- 设置 creator_id
- 初始化 record_status="正常"
- 设置 needs_review=FALSE

2. AI识别工作流
触发：当 company_name_raw 变更时
动作：
- 调用AI服务识别
- 填充 company_name_clean
- 设置 auto_fill_confidence
- 设置 needs_review（根据置信度）

3. 定时刷新工作流
触发：每日9:00
动作：
- 刷新 days_until_next
- 刷新 contact_count_month
- 检查SLA状态
- 触发逾期提醒

4. 审核提醒工作流
触发：当 needs_review=TRUE
动作：
- 发送通知给负责人
- 创建待办任务
- 设置审核期限
```

## 故障排查指南

### 常见问题

| 问题 | 可能原因 | 解决方案 |
|-----|---------|---------|
| record_id重复 | 导入数据唯一性冲突 | 检查导入数据，使用GUID生成器重新生成 |
| create_time时区错误 | 时区配置不一致 | 统一使用UTC+8，配置服务器时区 |
| AI识别失败 | 服务不可用/格式不支持 | 检查AI服务状态，确认数据格式 |
| 动态计算不更新 | 公式配置错误 | 检查公式语法，验证触发条件 |
| 关系映射失败 | 关联表无匹配数据 | 检查目标表数据，确认匹配条件 |

### 排查步骤

```
1. 定位问题字段
   - 确定是哪个字段出现问题
   - 检查字段类型和验证规则

2. 检查自动化状态
   - 查看自动化工作流执行日志
   - 确认触发条件是否满足

3. 验证数据质量
   - 检查原始数据格式
   - 确认必填字段是否完整

4. 测试自动化逻辑
   - 手动触发工作流
   - 查看执行结果和错误信息

5. 联系技术支持
   - 提供record_id和错误日志
   - 描述问题复现步骤
```

## 字段扩展指南

### 扩展原则

| 原则 | 说明 |
|-----|------|
| 兼容现有 | 新字段不破坏现有逻辑 |
| 分类归属 | 新字段归属现有分类或新建分类 |
| 文档同步 | 更新字段文档和使用说明 |
| 测试验证 | 充分测试后上线 |

### 扩展步骤

```
1. 需求分析
   - 确定新字段的业务需求
   - 选择合适的字段类型
   - 定义验证规则

2. 设计实现
   - 在测试环境创建字段
   - 配置相关公式和自动化
   - 测试各种场景

3. 上线部署
   - 在生产环境创建字段
   - 配置相同逻辑
   - 监控运行状态

4. 文档更新
   - 更新字段说明文档
   - 更新使用手册
   - 通知相关人员
```
