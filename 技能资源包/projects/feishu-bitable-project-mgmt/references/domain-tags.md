## 核心理念

**痛点**：53%项目因低估领域复杂度导致延期，资源错配严重。

**目标**：建立科学的复杂度评估体系，让资源配置精准匹配项目需求。

## 复杂度评估模型

### 专业领域复杂度公式

```formula
专业领域复杂度 = (
  {技术新颖度} × 0.25 + 
  {合规要求强度} × 0.2 + 
  {跨领域依赖数} × 0.2 + 
  {不确定性系数} × 0.15 + 
  {技能稀缺度} × 0.1 + 
  {失败成本} × 0.1
) × 100
```

### 各维度评估标准

| 维度 | 权重 | 评估标准 | 数据来源 |
|-----|-----|---------|---------|
| 技术新颖度 | 25% | 新技术比例×技术风险系数 | 技术栈标签 |
| 合规要求强度 | 20% | 法规数量×违规成本 | 合规矩阵 |
| 跨领域依赖数 | 20% | 依赖领域数量×依赖复杂度 | 依赖关系图 |
| 不确定性系数 | 15% | 需求变更频率×范围蔓延风险 | 历史数据 |
| 技能稀缺度 | 10% | 所需技能的市场稀缺度 | 技能库 |
| 失败成本 | 10% | 失败影响范围×恢复难度 | 业务影响评估 |

### 复杂度等级

| 复杂度指数 | 等级 | 标签 | 建议策略 |
|-----------|-----|------|---------|
| <30 | 低 | ★☆☆ | 快速迭代，敏捷开发 |
| 30-50 | 中低 | ★★☆ | 标准流程，适当缓冲 |
| 50-70 | 中 | ★★★ | 结构化流程，充足缓冲 |
| 70-85 | 高 | ★★★★ | 严格管控，专家参与 |
| >85 | 极高 | ★★★★★ | 分阶段交付，风险隔离 |

## 标签应用场景

### 场景1：项目立项评估

**输入**：新项目"智慧医疗影像诊断平台"

**标签识别结果**：
```
领域标签：医疗健康 - 医疗器械
核心标签：🔴隐私保护 | 🟡临床验证 | 🔴监管审批

复杂度评估：
- 技术新颖度：75（AI+医疗是新领域）
- 合规要求：95（需NMPA三类认证）
- 跨领域依赖：85（医疗+AI+IT多领域）
- 不确定性：60（临床验证结果不确定）
- 技能稀缺度：80（复合型人才稀缺）
- 失败成本：90（涉及患者安全）

综合复杂度指数：82（★★★★☆ 高）

适配策略：
1. 开发周期 = 标准周期 × 1.6
2. 强制分阶段交付（每个阶段可独立验收）
3. 预留合规专家资源
4. 建立监管沙盒环境进行验证
```

### 场景2：资源配置优化

**输入**：现有项目资源冲突

**标签辅助决策**：
```
项目A：智能制造 - 设备预测维护
- 标签：🔴设备集成 | 🟡实时数据 | ★★★☆☆
- 当前状态：复杂度匹配良好

项目B：新零售 - 全渠道库存
- 标签：🔴流量峰值 | 🟡实时库存 | ★★★☆☆
- 当前状态：需要大促期间资源

资源冲突：两个项目同时争抢基础设施团队

决策建议：
基于标签匹配度和业务优先级，优先保障项目B：
- 项目B涉及大促（时间敏感）
- 项目A可延后2周（产线改造窗口可协调）
- 冲突风险：项目A降低10% → 可接受
```

### 场景3：风险预警触发

**输入**：项目执行中出现异常

**标签辅助分析**：
```
项目：金融科技 - 智能投顾系统
标签：🔴高合规 | 🔴数据敏感 | ★★★★☆

异常事件：合规部门要求增加审计模块

风险评估：
- 合规要求变更 → 合规要求强度 +15%
- 新增模块 → 跨领域依赖 +10%
- 预计延期 → 时间紧迫度 +20%

更新后复杂度指数：85 → 92（★★★★★ 极高）

预警触发：
⚠️ 复杂度已达到极高等级，建议：
1. 重新评估项目范围
2. 申请追加资源和时间
3. 考虑分阶段交付可能性
4. 启动风险应对预案
```
