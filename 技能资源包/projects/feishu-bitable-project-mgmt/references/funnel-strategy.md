## 策略概述

### 核心思想

| 策略 | 说明 |
|-----|------|
| 放弃全覆盖 | 不要试图穷举所有可能性，那会得到"屎山"规则库 |
| 追求动态进化 | 业务是动态的，系统也要动态进化 |
| 不漏信息 | 用AI兜底确保未知场景不遗漏 |

### 漏斗模型

```
高频核心场景 (20%)  ──────────────────────► 硬规则  ──► 100%自动化
        │                                    │
        ▼                                    ▼
低频长尾场景 (60%)  ──────────────────────► AI兜底 ──► 待办人工处理
        │                                    │
        ▼                                    ▼
未知新增场景 (20%)  ──────────────────────► 复核池  ──► 规则挖矿
                                                           │
                                                           ▼
                                                    新规则孵化
```

## 第一步：二八定律锁定核心

### 操作方法

```
1. 拉取数据
   - 过去3个月的聊天记录或工单记录
   - 导出为Excel/CSV格式

2. 统计频次
   - 统计关键词组合出现频率
   - 找出Top10高频组合
   - 示例：
     "合同审批" - 156次
     "报价申请" - 98次
     "服务器报警" - 87次

3. 只针对Top10场景写规则
   - 优先覆盖最高频的场景
   - 20%的场景解决80%的重复劳动
```

### 示例：高频场景统计表

| 序号 | 关键词组合 | 频次 | 累计占比 | 是否覆盖 |
|-----|-----------|------|---------|---------|
| 1 | 合同审批 | 156 | 15.6% | ✓ |
| 2 | 报价申请 | 98 | 25.4% | ✓ |
| 3 | 服务器报警 | 87 | 34.1% | ✓ |
| 4 | 客户投诉 | 76 | 41.7% | ✓ |
| 5 | 库存预警 | 65 | 48.2% | ✓ |
| 6 | 预算超支 | 54 | 53.6% | 待定 |
| 7 | 人员请假 | 48 | 58.4% | 待定 |
| 8 | 采购申请 | 43 | 62.7% | 待定 |
| 9 | 发票核销 | 38 | 66.5% | - |
| 10 | 资质审核 | 35 | 70.0% | - |

### 为什么只覆盖20%

| 原因 | 说明 |
|-----|------|
| 边际效益递减 | 第11个场景可能只占总体的2%，投入产出比低 |
| 长尾处理成本低 | 低频场景人工处理成本远低于开发规则 |
| 快速见效 | 专注高频场景可快速看到效果 |

## 第二步：同义词库解决表达差异

### 问题场景

```
规则写的是："合同审批"
用户说的："签合同"、"法务看合同"、"合同盖章"、"审批流程"
→ 硬规则会失效
```

### 解决方案：意图映射表

```
┌─────────────────────────────────────────────────────────────┐
│  标准意图        │  同义词/变体                              │
├─────────────────────────────────────────────────────────────┤
│  合同审核        │  签合同、法务看合同、合同盖章、审批流程   │
│  报价申请        │  申请报价、价格确认、询价、报价单         │
│  服务器报警      │  服务器挂了、机器报警、系统异常、宕机     │
│  客户投诉        │  用户反馈、客诉、用户不满、投诉反馈       │
└─────────────────────────────────────────────────────────────┘
```

### 规则配置改写

```formula
# 错误的写法
IF 消息包含 "合同审批"

# 正确的写法
IF 消息包含 (
  查找「意图映射表」中标准意图为"合同审核"的所有变体
)
```

### 效果

- 无论用户怎么变着法子说
- 只要命中词库里的任意变体
- 都能归拢到同一个标准场景

## 第三步：AI兜底机制

### 为什么需要AI兜底

```
规则无法穷举所有可能性
长尾场景（60%）无法用硬规则覆盖
必须用AI语义理解来兜底
```

### AI兜底流程

```
┌─────────────────────────────────────────────────────────────┐
│                    消息进入                                  │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              硬规则匹配                                      │
│         检查是否命中已知规则                                 │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
        ┌───────────┐                   ┌───────────┐
        │  命中规则  │                   │ 未命中规则 │
        └───────────┘                   └───────────┘
              │                               │
              ▼                               ▼
        ┌───────────┐                   ┌───────────┐
        │ 执行规则   │                   │  发送给AI  │
        └───────────┘                   └───────────┘
                                               │
                                               ▼
                                    ┌─────────────────┐
                                    │ AI分析消息意图   │
                                    │ - 业务相关?      │
                                    │ - 关键信息提取?  │
                                    │ - 闲聊忽略?      │
                                    └─────────────────┘
                                               │
                                               ▼
                                    ┌─────────────────┐
                                    │ 推送到复核池     │
                                    │ 「待分类」       │
                                    └─────────────────┘
```

### AI提示词模板

```
请分析这条消息的意图：

消息内容：{message}

分析要求：
1. 判断是否为业务相关（是/否）
2. 如果是业务相关，提取关键信息
3. 标记消息类型（待分类）
4. 如果是闲聊，直接忽略

输出格式：
- 业务相关：是/否
- 消息类型：{类型}
- 关键信息：{提取内容}
- 建议动作：{下一步}
```

## 第四步：规则挖矿机制

### 核心理念

```
本周的"未知场景" → 人工处理 → 下周变成"已知规则"
覆盖率会从20%每月增长5%-10%
```

### 挖矿流程

```
┌─────────────────────────────────────────────────────────────┐
│                    每周五人工复核                            │
│               查看「人工复核池」里的数据                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                    寻找规律                                  │
│         最近是否有5条同类消息没有规则?                        │
└─────────────────────────────────────────────────────────────┘
                              │
              ┌───────────────┴───────────────┐
              ▼                               ▼
        ┌───────────┐                   ┌───────────┐
        │  找到规律  │                   │  未找到规律 │
        └───────────┘                   └───────────┘
              │                               │
              ▼                               ▼
        ┌───────────┐                   ┌───────────┐
        │ 立即创建规则│                  │  继续观察  │
        │  加入意图表 │                  │  下周再查  │
        └───────────┘                   └───────────┘
```

### 挖矿触发条件

| 条件 | 说明 |
|-----|------|
| 同类消息≥5条 | 证明是规律而非偶然 |
| 无现有规则 | 不是规则漏匹配 |
| 有人工处理价值 | 值得自动化 |

### 规则创建步骤

```
1. 提取共同特征
   - 消息中的共同关键词
   - 共同的执行动作

2. 创建新规则
   - 定义触发条件
   - 定义操作步骤
   - 定义关联字段

3. 加入意图映射表
   - 添加标准意图
   - 添加同义词变体

4. 测试验证
   - 用历史消息测试
   - 确认无误后启用
```

## 验收标准

| 场景类型 | 覆盖策略 | 你的动作 | 目标 |
|---------|---------|---------|------|
| 高频核心(20%) | 硬规则+同义词库 | 提取历史数据，配置精准规则 | 100%自动化，零人工 |
| 低频长尾(60%) | AI语义兜底 | 配置"未匹配转AI"流程 | 100%捕获，推送待办 |
| 未知新增(20%) | 人工复核+规则迭代 | 每周查看复核池，提取新规则 | 持续进化 |

### 关键指标

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| 覆盖率 | 每月+5-10% | 从20%开始增长 |
| 自动化率 | >80% | 高频场景100% |
| 漏报率 | <1% | AI兜底确保 |
| 规则质量 | 修正率<5% | 每条规则准确 |

## 实施检查清单

### 启动阶段（第一周）

```
□ 拉取过去3个月聊天/工单记录
□ 统计高频关键词组合
□ 确定Top10高频场景
□ 配置Top10的硬规则
□ 配置Top10的同义词库
□ 配置AI兜底流程
□ 创建人工复核池
```

### 运行阶段（每周）

```
□ 周五查看复核池
□ 寻找规律，提取新规则
□ 更新意图映射表
□ 测试验证新规则
□ 月底复盘覆盖率增长
```

### 优化阶段（每月）

```
□ 分析修正记录
□ 优化低质量规则
□ 补充遗漏的同义词
□ 评估是否扩展高频场景范围
```
