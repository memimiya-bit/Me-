## 流程概述

### 销售流程全景

```
商机识别 → 客户开发 → 订单转化 → 生产执行 → 售后服务
    ↓          ↓          ↓          ↓          ↓
  商机评估   需求对接   价格审批   交期确认   满意度跟踪
```

### 服装企业销售特点

- 季节性强：春夏/秋冬两季为主要销售周期
- 批量订单：通常以 SKU 为单位的批量订单
- 定制需求：品牌客户可能有定制化要求
- 交期敏感：服装销售季节性要求交期精准

## 流程节点映射

### 节点一：商机识别

**触发时机**：发现潜在客户需求
**关键动作**：
- 录入客户基本信息
- 评估商机价值
- 判断是否跟进

**输出**：商机记录

### 节点二：客户开发

**触发时机**：确认商机有效
**关键动作**：
- 深挖客户需求
- 提供产品方案
- 进行商务洽谈

**输出**：需求确认书

### 节点三：订单转化

**触发时机**：客户确认报价
**关键动作**：
- 价格审批
- 交期确认
- 合同签订

**输出**：正式订单

### 节点四：生产执行

**触发时机**：订单确认
**关键动作**：
- 纳入排产计划
- 生产进度跟踪
- 质量检验

**输出**：出货通知

### 节点五：售后服务

**触发时机**：客户收货
**关键动作**：
- 满意度调查
- 问题反馈处理
- 二次销售跟进

**输出**：客户评级更新

## 字段配置

### 销售流程必填字段

| 字段名称 | 类型 | 验证规则 | 说明 |
|---------|------|---------|------|
| 商机ID | 文本 | 唯一 | 自动生成，格式：OPP-YYYYMMDD-XXX |
| 客户名称 | 文本 | 必填 | 客户公司名称 |
| 客户等级 | 单选 | - | VIP/重要/普通/潜在 |
| 联系人 | 文本 | 必填 | 对接人姓名 |
| 联系电话 | 文本 | 手机号格式 | 11位手机号 |
| 邮箱 | 文本 | 邮箱格式 | 标准邮箱格式 |
| 销售阶段 | 单选 | - | 见下方选项 |
| 预计成交金额 | 数字 | 正数 | 单位：元 |
| 转化概率 | 数字 | 0-100% | 百分比 |
| 预计成交日期 | 日期 | - | 预计签单日期 |
| 责任人 | 人员 | 必填 | 销售负责人 |
| 销售类型 | 单选 | - | OEM/ODM/现货/定制 |
| 产品品类 | 多选 | - | 上装/下装/套装/配饰 |
| 数量范围 | 文本 | - | 如：500-1000件 |
| 单价区间 | 文本 | - | 如：50-100元 |
| 竞争对手 | 文本 | - | 已知竞争对手 |
| 风险等级 | 单选 | - | 高/中/低 |
| 下次跟进日期 | 日期 | - | AI自动计算 |
| 跟进记录 | 富文本 | - | 历史跟进记录 |
| 备注 | 长文本 | - | 其他说明 |

### 销售阶段选项

| 阶段 | 名称 | 转化概率默认值 | 说明 |
|-----|------|--------------|------|
| 1 | 商机识别 | 10% | 刚发现商机 |
| 2 | 需求确认 | 25% | 需求已明确 |
| 3 | 方案提交 | 40% | 方案已发送 |
| 4 | 商务谈判 | 60% | 正在谈价格 |
| 5 | 合同审批 | 80% | 合同审核中 |
| 6 | 订单签订 | 90% | 即将签单 |
| 7 | 已成交 | 100% | 合同已签 |
| 8 | 输单 | 0% | 商机流失 |

### 风险预警规则

```公式
交期风险 = IF(AND(预计成交日期 < TODAY() + 7, 销售阶段 < 6), "高风险", "正常")
金额风险 = IF(预计成交金额 > 1000000, "高价值", "普通")
竞争风险 = IF(竞争对手 ≠ "", "有竞争", "无竞争")
综合风险 = IF(OR(交期风险="高风险", 金额风险="高价值"), "需要关注", "正常跟进")
```

## 工作流配置

### 工作流一：商机自动分配

```
触发：当商机ID创建时
条件：责任人未指定
动作：
  1. 根据客户等级自动分配：
     - VIP客户 → 高级销售经理
     - 重要客户 → 资深销售
     - 普通客户 → 初级销售
  2. 发送通知给责任人
```

### 工作流二：阶段变更提醒

```
触发：当销售阶段变更时
动作：
  1. 记录变更历史
  2. 如果阶段=商务谈判：
     - 提醒责任人准备报价单
     - 提醒财务准备价格审批
  3. 如果阶段=合同审批：
     - 通知法务审核
     - 通知财务确认付款方式
```

### 工作流三：跟进超时预警

```
触发：每天检查
条件：下次跟进日期 < TODAY() AND 销售阶段 < 7
动作：
  1. 发送提醒给责任人
  2. 标记为"待跟进"
  3. 如果超时超过3天：
     - 升级通知给销售经理
```

### 工作流四：成交自动统计

```
触发：当销售阶段变更为"已成交"
动作：
  1. 更新成交日期
  2. 同步到客户台账的成交记录
  3. 发送成交喜报
  4. 触发客户评级更新逻辑
```

## 可视化图表

### 图表一：销售漏斗

**类型**：漏斗图
**数据维度**：
- X轴：销售阶段
- Y轴：商机数量/金额
**筛选**：本季度/本年度

### 图表二：销售阶段分布

**类型**：环形图
**数据维度**：各阶段商机数量占比
**颜色**：按阶段递进配色

### 图表三：销售趋势

**类型**：折线图
**数据维度**：按月度统计成交金额
**对比**：同期对比线

### 图表四：客户等级分布

**类型**：柱状图
**数据维度**：按客户等级统计商机数量和金额

### 图表五：责任人业绩排行

**类型**：排名表
**数据维度**：按成交金额排名前10
**显示**：成交数、成交金额、完成率
