## 核心理念

**痛点**：通用AI助手不了解岗位专业语言，产品经理收到"Java内存泄漏"预警毫无意义。

**目标**：让每个岗位的人看到"为他们定制"的AI响应，用他们熟悉的语言讲述他们关心的价值。

## 智能适配机制

### 角色识别

```formula
用户角色 = IF(
  ISBLANK({用户岗位}),
  AI_PREDICT({历史行为}, "预测最可能的岗位"),
  {用户岗位}
)
```

### 智能适配公式

```formula
AI响应模式 = MATCH(
  {当前用户角色},
  岗位能力模型,
  AI_GENERATE(
    "用{角色专业语言}解释{原始数据}，
     聚焦{角色KPI目标}，
     给出{角色}可执行的建议"
  )
)
```

### 适配决策树

```
原始数据输入
      │
      ▼
识别用户角色 ──→ 查询岗位知识图谱
      │
      ▼
获取角色关注指标和痛点
      │
      ▼
翻译原始数据 ──→ 用角色语言重构
      │
      ▼
聚焦角色关切 ──→ 强调与角色相关的价值点
      │
      ▼
生成行动建议 ──→ 给出角色可执行的建议
      │
      ▼
输出定制化报告
```

### 响应模板库

**通用模板**：
```
【{日期} {项目名称} {角色}视角报告】

核心结论：{一句话总结，用角色语言}

{角色}关注要点：
• {关键指标1}：{数值}（{同比/环比变化}）
• {关键指标2}：{数值}（{同比/环比变化}）
• {关键指标3}：{数值}（{同比/环比变化}）

{与角色利益相关的说明}

行动建议：
1. {建议1} → 预期效果：{效果}
2. {建议2} → 预期效果：{效果}
3. {建议3} → 预期效果：{效果}

需要关注的风险：
• {风险1}：{影响说明}
• {风险2}：{影响说明}
```

| 技术经理 | "数据库连接池耗尽，当前线程阻塞增长300%。建议：1. 扩容连接池128→256 2. 优化慢查询 3. 考虑读写分离。预计P99延迟从850ms降至200ms。" |
| 财务BP | "系统延迟导致的收入损失约18万/天，月度影响约540万。建议评估扩容投入ROI，10万/月的扩容成本可避免约540万/月的潜在损失，ROI=54x。" |
| COO | "支付流程处理效率下降38%，当前P99延迟850ms。建议：1. 启动应急扩容 2. 限流保护核心链路 3. 准备客诉应对方案。预计2小时内恢复。" |
