# 服装行业知识文档数据层规划技能

## 任务目标

本技能用于帮助用户完成服装行业知识文档数据库设计，实现非结构化文档转化为可关联、可追溯、可复用的知识资产，与项目节点库和协作库形成"业务执行-协同时效-知识沉淀"三位一体架构。

**核心能力包含：**
- 知识资产核心表设计：构建术语库和专家知识库，实现隐性知识显性化
- 文档流转中枢表设计：实现跨组织文档传递的完整追踪和关键动作闭环
- 业务场景知识包设计：封装可复用的知识解决方案，支持自动推荐
- 三库联动机制：打通知识库与项目节点库、协作库的关联关系

**触发条件：**
- 用户需要"规划服装行业知识文档数据层"
- 用户提到"术语库"、"知识沉淀"、"专家经验"
- 用户需要"管理工艺文档"、"跨组织文档流转"
- 用户询问"如何复用历史解决方案"、"知识自动推荐"

### 三大痛点与解决

| 痛点 | 说明 | 解决方案 |
|------|------|----------|
| 专业术语混乱 | 各部门自定义词汇表，跨境沟通歧义 | 统一术语ID体系（FAB-001） |
| 跨组织文档断层 | 依赖邮件/微信碎片化传递 | 完整流转追踪 from→to |
| 隐性知识流失 | 老师傅经验随离职流失 | expert_knowledge 显性化 |

### 行业特性强化

| 特性 | 说明 | 应用场景 |
|------|------|----------|
| 多语言映射 | 解决跨境供应链术语歧义 | 意大利语 elasticità/stretch |
| 季节性生命周期 | retention_policy 匹配季节+2年 | 工艺单保存规范 |
| 向量检索 | 语义搜索快速定位历史方案 | 输入"领口卷边"匹配修正案例 |
| 多国合规 | REACH vs CPSC 冲突标注 | ZDHC MRSL 合规检查 |

## 工作流程

### 步骤一：知识域识别

确认需要规划的知识管理领域：

| 知识域 | 核心表 | 适用场景 |
|--------|--------|----------|
| 术语管理 | term_glossary | 专业词汇统一、跨境沟通 |
| 专家知识 | expert_knowledge | 隐性知识显性化、经验复用 |
| 文档管理 | doc_master, doc_flow | 工艺单流转、合规追踪 |
| 版本控制 | version_control | 工艺变更溯源 |
| 知识封装 | knowledge_package | 场景化知识复用 |
| 表单模板 | form_template | 标准表单生成 |

### 步骤二：术语编码体系配置

配置术语分类编码规则：

| 前缀 | 类别 | 示例 |
|------|------|------|
| FAB | 面料类 | FAB-001=克重, FAB-002=纱支 |
| TECH | 工艺类 | TECH-001=洗水工艺, TECH-002=印花 |
| RSL | 环保类 | RSL-001=AZO染料, RSL-002=塑化剂 |
| QC | 质检类 | QC-001=色牢度, QC-002=缩水率 |
| REG | 合规类 | REG-001=REACH, REG-002=CPSC |

### 步骤三：知识来源分类

配置隐性知识显性化来源：

| 来源类型 | 说明 | 验证要求 |
|----------|------|----------|
| 老师傅口述 | 资深工匠经验 | 需3个validation_cases |
| 事故报告 | 历史问题总结 | 需root_cause分析 |
| 设计笔记 | 设计师原创思路 | 需项目关联 |
| 测试报告 | 实验数据积累 | 需测试方法说明 |

### 步骤四：文档生命周期配置

配置季节性文档保留策略：

| 文档类型 | 保留周期 | 说明 |
|----------|----------|------|
| 工艺单 | season+2年 | 过季后再保留2年 |
| 质检报告 | 3年 | 品质追溯用 |
| 环保证书 | 5年 | 合规长期保存 |
| 合同协议 | 10年 | 法律追溯 |

### 步骤五：三库联动配置

配置知识库与项目、协作库的关联：

| 联动场景 | 触发条件 | 联动结果 |
|----------|----------|----------|
| 项目创建 | project_type=羽绒服 | 推送RSL合规包 |
| 质检争议 | doc_flow.status=驳回 | 检索expert_knowledge |
| 协议终止 | change_reason=环保违规 | 锁定关联知识包 |

## 使用示例

### 示例一：术语库数据层

**场景：** 用户需要构建服装行业统一术语库

**输入：**
- 知识域：术语管理
- 术语分类：面料/工艺/RSL/QC

**规划流程：**
1. term_glossary：统一术语ID体系
2. 配置多语言映射（zh/en/zh-HK）
3. 关联行业标准（ISO/GB/AATCC）
4. 建立术语关系网络

**输出：** 术语编码体系 + 多语言对照 + 标准关联

### 示例二：工艺单流转数据层

**场景：** 用户需要管理跨组织工艺单流转

**输入：**
- 知识域：文档管理 + 版本控制
- 流转路径：品牌设计组→代工厂版房→QC部门

**规划流程：**
1. doc_master：工艺单生命周期管理
2. doc_flow：关键动作闭环（色牢度确认）
3. version_control：工艺变更多角色会签
4. 关联合规检查（RSL标准）

**输出：** 完整流转追踪 + 合规检查 + 版本溯源

### 示例三：专家知识数据层

**场景：** 用户需要沉淀老师傅的隐性经验

**输入：**
- 知识域：专家知识 + 知识包
- 核心需求：经验复用、自动推荐

**规划流程：**
1. expert_knowledge：建立知识来源分类
2. 配置向量检索（语义搜索）
3. knowledge_package：封装解决方案
4. 设置自动推荐规则（项目类型匹配）

**输出：** 专家知识库 + 向量索引 + 自动推荐

## 注意事项

- 本技能专注于知识文档领域，与 apparel-industry-data-planning（通用数据层）和 apparel-collab-planning（项目协作）形成三位一体
- 术语编码体系需与企业内部编码规则保持一致
- 隐性知识显性化需与业务专家配合完成
- 向量检索需要向量数据库支持（如 Milvus、Pinecone）
- 多国合规冲突标注需与法务团队确认
- 建议三库组合使用获取完整服装行业数据层解决方案
