## 字段类型说明

### 文本类型

适用于：标题、摘要、编号等短文本

**字段配置示例：**
```json
{
  "field_name": "内容摘要",
  "type": 1,
  "property": {
    "max_length": 200
  }
}
```

### 长文本类型

适用于：详细内容、描述、备注等长内容

**字段配置示例：**
```json
{
  "field_name": "详细内容",
  "type": 2,
  "property": {
    "rich_text": true
  }
}
```

### 单选类型

适用于：状态、类型、等级等有限选项

**字段配置示例：**
```json
{
  "field_name": "处理状态",
  "type": 3,
  "property": {
    "options": [
      {"name": "待处理", "color": 0},
      {"name": "处理中", "color": 2},
      {"name": "已处理", "color": 4},
      {"name": "已分流", "color": 6}
    ]
  }
}
```

### 多选类型

适用于：标签、技能等多值选择

**字段配置示例：**
```json
{
  "field_name": "关键词标签",
  "type": 4,
  "property": {
    "options": [
      {"name": "技术", "color": 0},
      {"name": "客户", "color": 2},
      {"name": "项目", "color": 4},
      {"name": "质量", "color": 6}
    ]
  }
}
```

### 数字类型

适用于：优先级、评分、数量等数值

**字段配置示例：**
```json
{
  "field_name": "优先级",
  "type": 5,
  "property": {
    "formatter": "0"
  }
}
```

### 日期类型

适用于：时间、截止日期等

**字段配置示例：**
```json
{
  "field_name": "创建时间",
  "type": 5,
  "property": {
    "date_formatter": "yyyy/MM/dd HH:mm"
  }
}
```

### 人员类型

适用于：负责人、执行人等

**字段配置示例：**
```json
{
  "field_name": "创建人",
  "type": 7
}
```

## 字段命名规范

### 命名原则

1. **简洁明确**：字段名应简洁明了，一看就知道存储什么内容
2. **统一格式**：使用中文命名，保持格式统一
3. **避免歧义**：相同含义的字段在不同表中使用相同名称
4. **业务导向**：字段名应符合业务用语习惯

### 命名示例

| 业务含义 | 推荐名称 | 避免使用 |
|----------|----------|----------|
| 负责人 | 负责人/处理人 | owner/handler |
| 创建时间 | 创建时间/录入时间 | createTime/ct |
| 处理状态 | 处理状态/状态 | status/s |
| 客户名称 | 客户名称/客户 | customer/c |
| 优先级 | 优先级/重要程度 | priority/p |
| 联系方式 | 联系方式/电话/邮箱 | contact/con |

### 常用字段前缀

| 前缀 | 含义 | 示例 |
|------|------|------|
| 处理 | 处理相关 | 处理状态、处理人、处理时间 |
| 关联 | 关联关系 | 关联项目、关联客户、关联订单 |
| 所属 | 归属关系 | 所属部门、所属项目、所属分类 |
| 预估 | 预计值 | 预估金额、预估工期、预估成本 |
| 实际 | 实际值 | 实际金额、实际工期、实际成本 |
