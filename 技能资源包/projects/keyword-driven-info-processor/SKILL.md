# 关键词驱动信息处理技能

## 任务目标

本技能用于实现智能信息分类与飞书多维表格的自动化更新，将散落在笔记、文档、对话中的关键信息自动提取、分流并写入目标表格。

**核心能力包含：**
- 关键词提取引擎：从用户输入中智能识别核心关键词
- 多源信息采集：从笔记、文档、对话上下文中提取相关信息
- 智能分流判断：根据关键词类型自动分类并确定目标表格
- 表格自动更新：将处理后的结构化数据写入飞书多维表格

**触发条件：**
- 用户明确表达需要"整理信息"、"分类数据"
- 用户提到"关键词"、"分流"、"写入表格"
- 用户需要"自动记录到飞书"、"更新多维表格"
- 用户希望"从笔记/文档提取内容"

## 前置准备

### 飞书授权配置
- 已完成飞书多维表格 OAuth 授权（credential: feishu_bitable）
- 目标多维表格已创建并获取 app_token
- 目标数据表已创建并获取 table_id

### 获取表格标识
1. 打开飞书多维表格
2. 复制浏览器地址栏 URL：`https://example.feishu.cn/base/{app_token}?table={table_id}`
3. 提取 app_token 和 table_id

### 表格字段设计
建议创建以下标准字段：

| 字段名称 | 字段类型 | 必填 | 说明 |
|----------|----------|------|------|
| 关键词类型 | 文本 | 是 | 识别的关键词分类 |
| 信息来源 | 单选 | 是 | 笔记/文档/对话 |
| 内容摘要 | 文本 | 是 | 50字以内的内容概括 |
| 详细内容 | 长文本 | 否 | 原始内容全文 |
| 创建时间 | 日期 | 是 | 自动记录 |
| 处理状态 | 单选 | 是 | 待处理/已处理/已分流 |
| 优先级 | 数字 | 否 | 1=高 2=中 3=低 |

详细字段设计见 [references/table-field-design.md](references/table-field-design.md)

## 工作流程

### 步骤一：关键词提取与分析

接收用户输入，使用 AI 能力提取：

1. **核心关键词**：识别 1-3 个关键词
2. **信息类型判断**：技术类/业务类/日常类/客户类/项目类
3. **优先级评估**：基于关键词重要性打分

**输出格式：**
```json
{
  "keywords": ["客户", "投诉", "质量"],
  "info_type": "客户类",
  "priority": 1,
  "summary": "客户投诉产品质量问题，需关注"
}
```

### 步骤二：智能分流判断

根据关键词匹配规则自动分流：

| 关键词类型 | 目标表格类型 | 优先级 |
|------------|--------------|--------|
| 技术/开发/代码/系统 | 技术文档表 | 1 |
| 客户/销售/订单/商机 | 销售管理表 | 2 |
| 项目/任务/里程碑 | 项目跟踪表 | 2 |
| 质量/投诉/反馈 | 客服处理表 | 1 |
| 财务/预算/成本 | 成本分析表 | 1 |
| 日常/其他 | 通用信息表 | 3 |

### 步骤三：数据写入飞书表格

调用脚本将数据写入目标多维表格：

**脚本调用示例：**
```bash
python scripts/feishu_table_updater.py \
  --app_token "YOUR_APP_TOKEN" \
  --table_id "YOUR_TABLE_ID" \
  --fields '{"关键词类型":"客户类","信息来源":"对话","内容摘要":"客户投诉产品质量","详细内容":"客户反馈衬衫存在色差问题","处理状态":"待处理","优先级":1}'
```

**返回结果：**
```json
{
  "status": "success",
  "record_id": "rec_xxxxx",
  "message": "数据已写入飞书表格"
}
```

### 步骤四：结果反馈

向用户确认处理结果：
- 成功：提示已写入的目标表格和信息摘要
- 失败：说明错误原因并建议重试

## 关键词分流规则

### 默认分流配置

```yaml
routing_rules:
  - keywords: ["技术", "开发", "代码", "系统", "API", "数据库"]
    target: "技术团队表格"
    priority: 1
    
  - keywords: ["客户", "销售", "订单", "商机", "报价"]
    target: "销售管理表格"
    priority: 2
    
  - keywords: ["项目", "任务", "里程碑", "进度"]
    target: "项目跟踪表格"
    priority: 2
    
  - keywords: ["投诉", "质量", "反馈", "问题", "售后"]
    target: "客服处理表格"
    priority: 1
    
  - keywords: ["预算", "成本", "财务", "报销", "支出"]
    target: "成本分析表格"
    priority: 1
    
  - keywords: ["会议", "讨论", "待办", "计划"]
    target: "日常事务表格"
    priority: 3
    
  - keywords: []
    target: "通用信息表格"
    priority: 3
```

### 自定义规则

用户可以通过以下方式扩展规则：
1. 直接提供关键词映射表
2. 说明业务场景让 AI 自动推断
3. 指定特定表格作为目标

## 使用示例

### 示例一：笔记信息整理

**场景/输入：**
用户上传会议笔记："今天讨论了新一代电商系统的架构设计，计划采用微服务架构，涉及用户系统、订单系统、支付系统三个核心模块，开发周期预计3个月。"

**处理流程：**
1. 提取关键词：技术/微服务/架构/开发
2. 判断类型：技术类
3. 分流目标：技术团队表格
4. 调用脚本写入飞书表格

**预期产出：**
- 写入技术团队表格
- 关键词类型：技术类
- 内容摘要：电商系统架构设计讨论
- 详细信息：包含微服务架构说明

### 示例二：客户对话记录

**场景/输入：**
用户输入："客户王总反馈上周下单的样品颜色有偏差，希望尽快处理，否则会影响后续合作。"

**处理流程：**
1. 提取关键词：客户/样品/颜色/偏差
2. 判断类型：客户类
3. 分流目标：客服处理表格
4. 优先级设为 1（高优先级）
5. 处理状态设为"待处理"

**预期产出：**
- 写入客服处理表格
- 关键词类型：客户类
- 优先级：1（高）
- 处理状态：待处理

### 示例三：项目任务记录

**场景/输入：**
用户输入："记录一下刚才讨论的新品开发项目，第一阶段需要完成市场调研和竞品分析，预计2周内完成。"

**处理流程：**
1. 提取关键词：项目/新品开发/市场调研
2. 判断类型：项目类
3. 分流目标：项目跟踪表格
4. 自动补充项目阶段信息

**预期产出：**
- 写入项目跟踪表格
- 关键词类型：项目类
- 内容摘要：新品开发项目第一阶段讨论
- 详细信息：包含市场调研和竞品分析任务

## 注意事项

- 首次使用需完成飞书授权配置
- 写入前需确认目标表格的 app_token 和 table_id 正确
- 建议单次写入不超过 100 条记录
- 大批量数据建议分批次写入（每批 50 条）
- 写入失败时检查网络连接和凭证有效性
- 关键词分流可根据业务需求自定义调整
- 敏感信息建议在写入前进行脱敏处理
