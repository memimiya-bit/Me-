## 数据完整性验证

### 主键验证

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| primary_key_uniqueness | 主键字段值必须全局唯一 | 错误 |
| primary_key_not_null | 主键字段不能为空 | 错误 |
| primary_key_format | 主键格式符合编码规范 | 警告 |

### 外键验证

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| foreign_key_existence | 外键字段值必须在目标表中存在 | 错误 |
| foreign_key_not_null | 非空外键不能为空 | 错误 |
| foreign_key_cascade | 级联删除/更新时保持引用完整性 | 错误 |

### 必填字段验证

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| required_field_validation | 标记为 required 的字段不能为空 | 错误 |
| conditional_required | 满足条件时必填的字段验证 | 警告 |

### 数据类型验证

| 规则 | 说明 | 严重级别 |
|------|------|----------|
| data_type_consistency | 数据类型必须符合字段定义 | 错误 |
| numeric_range | 数值在有效范围内 | 错误 |
| enum_value_valid | 枚举值在允许列表中 | 错误 |

## 业务规则验证

### 日期逻辑验证

| 规则 | 验证条件 | 严重级别 |
|------|----------|----------|
| end_after_start | 结束日期不能早于开始日期 | 错误 |
| actual_after_plan | 实际日期不能早于计划日期 | 警告 |
| effective_before_expiry | 生效日期不能晚于到期日期 | 错误 |
| duration_reasonable | 持续天数在合理范围内（如 1-3650 天） | 警告 |

### 数值范围验证

| 字段类型 | 验证规则 | 范围 | 严重级别 |
|----------|----------|------|----------|
| 百分比 | progress | 0-100 | 错误 |
| 风险评分 | risk_score | 1-10 | 错误 |
| 影响评分 | impact_score | 1-10 | 错误 |
| 评分 | rating | 0-5 | 错误 |
| 金额 | cur_xxx | >= 0 | 警告 |

### 状态转换验证

| 实体 | 允许的状态流转 |
|------|----------------|
| 项目 | 规划中 → 进行中 → 已完成/延期/已取消 |
| 任务 | 未开始 → 进行中 → 已完成/已延期/已取消 |
| 合同 | 起草中 → 审批中 → 已签订 → 执行中 → 已完成/已终止 |
| 风险 | 识别中 → 评估中 → 处理中 → 已化解/已升级 |

### 交叉字段验证

| 验证场景 | 验证规则 | 严重级别 |
|----------|----------|----------|
| 预算合理性 | cur_actual_cost > cur_budget * 2 时警告 | 警告 |
| 进度一致性 | progress > 0 但 dt_actual_start 为空时报错 | 错误 |
| 成本累计 | 项目实际成本 = 任务实际成本之和（允许 5%误差） | 警告 |
| 人员归属 | 任务执行人必须属于负责任部门 | 错误 |

## 性能优化建议

### 索引优化

| 优化类型 | 适用场景 | 建议 |
|----------|----------|------|
| 单字段索引 | 高频查询字段 | 为 where 条件常用字段添加索引 |
| 联合索引 | 多字段组合查询 | 按查询频率排序创建联合索引 |
| 前缀索引 | 长文本字段 | 对长文本字段使用前缀索引 |
| 部分索引 | 条件索引 | 仅对活跃数据建立索引 |

**索引建议评估：**

| 查询类型 | 建议索引字段 |
|----------|--------------|
| 按项目查任务 | (project_code, task_status) |
| 按部门查人员 | (dept_code, active_status) |
| 按状态查记录 | (project_status, dt_updated) |
| 按时间范围查 | (dt_start, dt_end) |

### 存储优化

| 优化策略 | 说明 | 适用场景 |
|----------|------|----------|
| 冷热数据分离 | 历史数据归档到冷存储 | 数据量超过 10 万条 |
| 重复数据去重 | 相同数据合并存储 | 多表存在重复字段 |
| 压缩存储 | 对大字段启用压缩 | 包含长文本的字段 |
| 分区表 | 按时间或业务维度分区 | 超大表（超过 100 万行） |

### 查询优化

| 优化建议 | 说明 |
|----------|------|
| 避免全表扫描 | where 条件必须命中索引 |
| 限制返回结果 | 使用 limit 分页加载 |
| 预计算汇总 | 对频繁聚合计算预存结果 |
| 延迟加载 | 大字段按需加载 |
| 缓存热点数据 | 常用查询结果缓存 |

## 数据质量检测

### 缺失值分析

| 检测规则 | 说明 | 处理建议 |
|----------|------|----------|
| 完全为空 | 整行无任何有效数据 | 删除或标记为垃圾数据 |
| 关键字段缺失 | 主键、必填字段为空 | 回溯补录或标记无效 |
| 非关键字段缺失 | 可选字段为空 | 标记为空值，分析原因 |

### 异常值检测

| 检测方法 | 说明 | 阈值示例 |
|----------|------|----------|
| 统计方法 | 基于历史数据分布检测 | 超出 3 倍标准差 |
| 业务规则 | 基于业务逻辑检测 | 金额为负数 |
| 枚举验证 | 值不在允许列表中 | 无效枚举值 |
| 格式验证 | 不符合预设格式 | 手机号格式错误 |

### 一致性检查

| 检查类型 | 说明 | 检测逻辑 |
|----------|------|----------|
| 表内一致性 | 同一表内字段逻辑一致 | 金额 = 数量 × 单价 |
| 跨表一致性 | 关联表间数据一致 | 项目总成本 = 任务成本之和 |
| 历史一致性 | 与历史数据趋势一致 | 环比波动超过 50% 需确认 |

## AI治理功能

### 数据质量报告生成

**输入：**
- 缺失值分布统计
- 异常值检测结果
- 一致性检查结果

**输出：**
```
数据质量报告
============
总体评分: 85/100

缺失值统计:
- departments表: 0.2% 缺失率
- employees表: 1.5% 缺失率（主要在 phone字段）

异常值统计:
- projects表: 3条进度异常（进度100%但状态未完成）
- 金额字段: 2条负值记录

一致性检查:
- 通过率: 98.5%
- 失败项: 5条成本累计不一致

建议措施:
1. [高优先级] 补充 employees.phone 缺失数据
2. [中优先级] 修正 projects 表状态不一致
3. [低优先级] 优化索引提升查询性能
```

### 智能补全建议

**触发条件：** 关键字段缺失

**补全策略：**

| 场景 | 补全方法 | 数据来源 |
|------|----------|----------|
| 负责人缺失 | 引用部门默认负责人 | manager_id from departments |
| 预算缺失 | 参考同类项目平均预算 | AVG(cur_budget) by project_type |
| 日期缺失 | 引用标准工期推算 | dt_start + 标准工期 |
| 名称缺失 | 从关联表复制 | 从关联表获取 |

**输出格式：**
```
智能补全建议
============
字段: project_manager_id
建议值: EMP003（张三维）
置信度: 85%
依据: 该员工为 responsible_dept 的部门负责人
操作: [采纳建议] [忽略] [手动输入]
```

### 异常预警

**预警类型：**

| 预警类型 | 检测条件 | 通知方式 |
|----------|----------|----------|
| 进度异常 | 进度 > 0 但 > 30 天无更新 | 弹窗 + 邮件 |
| 成本超支 | 实际成本 > 预算 80% | 弹窗 + 邮件 |
| 延期风险 | 当前日期 > 计划结束日期且未完成 | 每日推送 |
| 数据异常 | 检测到异常值时 | 日报汇总 |

## 性能目标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 复杂查询响应时间 | < 2 秒 | 含多表关联和聚合计算 |
| 万级数据加载时间 | < 5 秒 | 单表 10000 条记录 |
| 并发支持能力 | 50+ 用户 | 同时在线操作 |
| 数据一致性保证 | 99.9% | 引用完整性和业务规则 |
