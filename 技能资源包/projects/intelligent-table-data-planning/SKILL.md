# 智能表格数据层规划技能

## 任务目标

本技能用于帮助用户完成企业级多维表格的底层数据架构设计，确保数据的完整性、一致性和可扩展性。

**核心能力包含：**
- 主数据模型构建：自动识别业务主数据，构建企业级核心数据模型
- 跨表映射引擎：建立多表之间的数据映射关系，支持复杂业务逻辑
- 知识文档集成：将企业知识文档与数据表智能关联
- 字段标准化规范：统一字段命名和数据类型标准
- 数据血缘追踪：记录数据来源、转换和使用路径
- 智能索引优化：根据查询模式自动优化数据结构

**触发条件：**
- 用户明确表达需要"规划数据层"、"设计表结构"、"配置主数据"
- 用户上传业务需求文档，希望进行数据结构设计
- 用户询问"如何建立表关系"、"如何管理主数据"
- 用户需要将现有表格进行规范化或重构

## 前置准备

- 确认用户业务域（组织人事、产品管理、供应链、项目管理、客户关系、财务核算）
- 明确核心业务实体和业务流程
- 识别已有数据源和数据质量状况

## 工作流程

### 步骤一：业务域识别

引导用户选择需要规划的数据层业务域：

| 业务域 | 核心实体 | 典型场景 |
|--------|----------|----------|
| 组织人事 | 部门、人员、角色 | 组织架构管理、权限配置 |
| 产品管理 | 产品、物料、BOM | 商品档案、配方管理 |
| 供应链管理 | 供应商、采购、库存 | 供应商管理、采购流程 |
| 项目管理 | 项目、任务、里程碑 | 项目跟踪、资源调配 |
| 客户关系 | 客户、商机、合同 | 销售管理、客户服务 |
| 财务核算 | 账户、凭证、预算 | 成本管理、财务报表 |

**交互示例：**
```
请选择需要规划的数据层业务域：
1. 组织人事
2. 产品管理
3. 供应链管理
4. 项目管理
5. 客户关系
6. 财务核算
```

### 步骤二：主数据识别

根据业务域，自动识别核心主数据实体：

| 业务域 | 主数据实体 |
|--------|------------|
| 组织人事 | 部门主数据、人员主数据、角色主数据 |
| 产品管理 | 产品主数据、物料主数据、工艺主数据 |
| 供应链管理 | 供应商主数据、采购商主数据、物流商主数据 |
| 项目管理 | 项目主数据、任务主数据、资源主数据 |
| 客户关系 | 客户主数据、联系人主数据、合同主数据 |
| 财务核算 | 账户主数据、科目主数据、预算主数据 |

**主数据识别要点：**
- 核心主数据：需全局统一管理，被多个业务模块引用
- 业务主数据：仅在特定业务域内使用
- 交易数据：由业务操作产生，记录具体业务事件

### 步骤三：映射关系定义

选择需要建立的数据映射关系类型：

**映射类型说明：**

| 映射类型 | 说明 | 示例 |
|----------|------|------|
| 一对一映射 | 两个实体一一对应 | 项目负责人 ←→ 人员主数据 |
| 一对多映射 | 一个实体对应多个实体 | 部门 ←→ 多个项目 |
| 多对多映射 | 实体间交叉对应 | 人员 ←→ 多个项目 |
| 计算映射 | 基于字段计算得出 | 项目总成本 = 人力+物料+外协 |
| 知识关联映射 | 数据与知识文档关联 | 项目记录 ←→ 项目知识包 |

### 步骤四：知识集成配置

选择知识文档集成深度：

| 集成级别 | 功能 | 适用场景 |
|----------|------|----------|
| 不集成 | 纯数据表，无知识关联 | 简单数据记录 |
| 基础集成 | 字段级附件关联 | 文档附件管理 |
| 深度集成 | 智能推荐+双向链接 | 知识密集型业务 |

**知识关联方式：**
- 字段级关联：特定字段直接关联文档（如质量标准字段 ←→ 检验标准文档）
- 记录级关联：整条记录关联知识包（如项目 ←→ SOP+合同+案例）
- 条件级关联：满足条件时自动推荐知识（如风险等级 > 8 → 风险处理指南）

### 步骤五：生成确认与输出

汇总规划方案，确认后生成：

1. **核心表清单**：包含表名、描述、主键、索引
2. **字段定义**：包含字段名、类型、约束、默认值
3. **映射关系图**：可视化展示表间关系
4. **数据血缘追踪方案**：记录数据来源和使用路径
5. **验证规则集**：完整性、一致性、范围校验规则

## 字段命名规范

### 标准前缀

| 字段类别 | 命名格式 | 示例 |
|----------|----------|------|
| ID字段 | 实体类型_code | project_code, dept_code |
| 名称字段 | 实体类型_name | project_name, dept_name |
| 状态字段 | 实体类型_status | project_status, order_status |
| 金额字段 | cur_金额描述 | cur_budget, cur_actual_cost |
| 时间字段 | dt_时间描述 | dt_start, dt_end, dt_plan |
| 百分比字段 | pct_描述 | pct_progress, pct_completion |
| 标识字段 | is_描述 | is_active, is_deleted |

### 命名原则
- 使用小写字母和下划线
- 避免缩写，使用完整单词
- 保持语义清晰，见名知意
- 主键统一使用 `xxx_code` 或 `xxx_id` 格式

## 数据类型标准

### 标识类
- `text`：通用文本，主键和普通标识
- `auto_increment`：自增ID（整数）
- `uuid`：全局唯一标识符

### 数值类
- `decimal(18,2)`：金额、预算等精确数值
- `decimal(10,1)`：工时、百分比等
- `integer`：计数、百分比（0-100）
- `float`：一般计算结果

### 时间类
- `date`：日期（年月日）
- `datetime`：日期时间（带时分秒）
- `timestamp`：时间戳（自动记录）

### 状态类
- `enum`：有限枚举值（如项目状态）
- `dropdown`：下拉选择（引用另一表）
- `multi_select`：多选标签

### 关系类
- `foreign_key`：外键引用
- `reference`：跨表引用
- `association`：多对多关联

## 使用示例

### 示例一：项目管理数据层规划

**场景：** 用户需要为服装企业规划项目管理系统数据层

**输入：**
- 业务域：项目管理
- 核心实体：项目、任务、资源、风险
- 合作企业：面料供应商、代工厂、品牌客户

**规划流程：**
1. 识别主数据：项目主数据、部门主数据、人员主数据、企业主数据
2. 设计核心表：projects、project_tasks、project_risks、departments、enterprises
3. 配置映射：responsible_dept → departments.dept_code
4. 添加知识集成：项目知识包（流程文档+合同模板+历史案例）

**输出：** 5张核心表 + 8个映射关系 + 完整字段定义

### 示例二：供应链数据层规划

**场景：** 用户需要规划供应商管理数据层

**输入：**
- 业务域：供应链管理
- 核心实体：供应商、物料、采购订单、库存

**规划流程：**
1. 识别主数据：企业主数据（供应商）、产品主数据（物料）
2. 设计核心表：enterprises、products、purchase_orders、inventory
3. 配置映射：supplier_code → enterprises.enterprise_code
4. 添加计算映射：采购金额 = 单价 × 数量

### 示例三：组织人事数据层规划

**场景：** 用户需要规划企业组织架构管理数据层

**输入：**
- 业务域：组织人事
- 核心实体：部门、人员、角色、技能

**规划流程：**
1. 识别主数据：部门主数据、人员主数据、角色主数据
2. 设计核心表：departments、employees、employee_roles
3. 配置层级映射：parent_dept_code 实现树形结构
4. 配置人员-角色多对多映射

## 注意事项

- 本技能专注于数据层架构设计，不涉及前端界面设计
- 优先识别核心主数据，确保数据一致性基础
- 映射关系设计需考虑业务扩展性
- 知识集成根据实际需求选择集成深度
- 生成方案后建议用户确认关键表结构和映射关系
- 复杂业务场景建议分步骤规划，先核心后扩展
