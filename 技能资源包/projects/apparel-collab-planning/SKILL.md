# 服装行业项目协作数据层规划技能

## 任务目标

本技能用于帮助用户完成服装行业专用数据层设计，聚焦项目节点管理和跨组织协同时效性，确保数据结构能精准支撑服装行业特有的阶段性协作特征。

**核心能力包含：**
- 项目节点数据库设计：以产品季/项目周期为单位，绑定产业链节点与业务实体
- 跨组织协作数据库设计：管理跨部门、跨企业的协作协议与时效追踪
- 季节性数据隔离：支持季节代码（SS24/AW25）替代传统时间戳
- 快反供应链支持：实时响应直播爆款等突发订单
- 牛鞭效应控制：追踪协作响应时效，识别供应链波动

**触发条件：**
- 用户需要"规划服装行业项目数据层"
- 用户提到"季节性项目"、"快反供应链"
- 用户需要"管理跨组织协作关系"
- 用户询问"如何追踪协作时效"、"牛鞭效应控制"
- 用户需要"项目节点与协作协议联动"

## 数据架构概览

### 双库联动结构

```
┌─────────────────────────────────────────────────────────────────┐
│                    project_node_db（项目节点库）                    │
│  project_master → node_mapping → project_bom                      │
│  以 season_code 为核心隔离维度                                    │
├─────────────────────────────────────────────────────────────────┤
│                    org_collab_db（组织协作库）                     │
│  org_collab_agreement → collab_instance → relationship_history   │
│  以 agreement_id 为核心管理时效                                    │
└─────────────────────────────────────────────────────────────────┘
```

### 数据库协同关系

| 协同场景 | 触发条件 | 联动结果 |
|----------|----------|----------|
| 快反供应链触发 | collab_instance.response_time > 4h | 自动创建快反项目 |
| 跨季成本分析 | 对比不同季节的 data_exchange_std | 成本差异分析 |
| ESG风险阻断 | relationship_history 含环保违规 | 锁定关联项目 |

## 设计原则

### 1. 季节性数据隔离

- 使用 season_code（SS24/AW25）替代传统时间戳
- 避免跨季数据混用（如将 AW24 面料误用于 SS25）
- 项目数据按季节归档，180天后移入冷库

### 2. 协同时效追踪

- 精确记录响应时间（快时尚要求 < 4h）
- 存储原始交互数据用于追溯
- 实时监控牛鞭效应指标

### 3. 多时区支持

- 所有时间字段存储 UTC 时间
- milestone_deadline 额外记录工厂本地时区
- 支持全球供应链协同（越南ICT、中国CST等）

### 4. ESG风险管控

- 关键字段包含 sustainability_score
- 环保违规自动触发项目锁定
- 支持 ZDHC 化学品追溯

## 工作流程

### 步骤一：业务域识别

确认需要规划的数据库类型：

| 数据库 | 适用场景 | 核心表 |
|--------|----------|--------|
| project_node_db | 项目周期管理、季节性隔离 | project_master, node_mapping, project_bom |
| org_collab_db | 跨组织协作、时效追踪 | org_collab_agreement, collab_instance, relationship_history |
| 双库联动 | 完整解决方案 | 全部6张表 |

### 步骤二：季节代码配置

配置项目季节代码规则：

| 季节代码 | 含义 | 生产周期 |
|----------|------|----------|
| SS | Spring/Summer 春夏 | 头年Q3开发，Q4生产，Q1上市 |
| AW | Autumn/Winter 秋冬 | 当年Q1开发，Q2生产，Q3上市 |
| FY | Full Year 全年款 | 无季节限制 |

### 步骤三：项目类型配置

配置项目类型区分：

| 项目类型 | 说明 | 时效要求 |
|----------|------|----------|
| 主线款 | 常规季节性产品 | 标准交期 |
| 快反款 | 直播爆款、追单 | 响应 < 4h |
| 联名款 | 品牌联名合作 | 特殊交期 |
| 特卖款 | 季末清仓 | 长周期 |

### 步骤四：协作协议配置

配置跨组织协作类型：

| 协作类型 | 说明 | 典型场景 |
|----------|------|----------|
| 订单协同 | 品牌商→代工厂的PO传输 | ERP对接 |
| 库存共享 | 门店-中央仓实时调拨 | ZARA模式 |
| 设计共创 | 多方联合设计开发 | 联名款 |
| 质量共建 | 品质标准共享 | 优衣库模式 |

### 步骤五：时效阈值配置

配置关键时效指标：

| 指标 | 快反阈值 | 标准阈值 | 告警规则 |
|------|----------|----------|----------|
| 协作响应时间 | < 4h | < 24h | 超时触发预警 |
| 样衣确认 | < 48h | < 7d | 延期影响大货 |
| 大货交期 | 按协议 | 按协议 | 违约追溯 |

## 使用示例

### 示例一：快反供应链数据层

**场景：** 用户需要为直播电商团队规划快反供应链数据层

**输入：**
- 数据库类型：双库联动
- 核心需求：直播爆款快速响应、跨组织时效追踪

**规划流程：**
1. project_node_db：
   - project_master（project_type=快反款）
   - node_mapping（快反协作节点）
   - project_bom（简化BOM）
2. org_collab_db：
   - org_collab_agreement（紧急补货协议）
   - collab_instance（响应时间追踪）
   - relationship_history（临时关系管理）

**输出：** 6张核心表 + 快反时效配置 + 预警规则

### 示例二：季节性项目数据层

**场景：** 用户需要规划 SS24 季节项目数据层

**输入：**
- 数据库类型：project_node_db
- 季节代码：SS24
- 项目类型：主线款

**规划流程：**
1. 创建 SS24 季节项目
2. 配置中游主责方（上波研发部）
3. 配置上游协作方（棉花供应商）
4. 设置里程碑（样衣确认、大货交期）
5. 冻结 SS24 BOM 配置

**输出：** 季节隔离的项目数据 + 里程碑预警配置

### 示例三：跨组织协作数据层

**场景：** 用户需要管理品牌商与代工厂的协作协议

**输入：**
- 数据库类型：org_collab_db
- 协作类型：订单协同
- 时效要求：响应 < 24h

**规划流程：**
1. 创建协作协议（from品牌部→to代工厂）
2. 配置数据交换标准（API对接）
3. 记录协作实例（每次PO交互）
4. 管理关系历史（协议变更）

**输出：** 协作协议管理 + 时效追踪 + 历史追溯

## 注意事项

- 本技能专注于项目节点和协作时效领域，与通用服装数据层规划（apparel-industry-data-planning）形成互补
- 季节代码需与企业内部编码规则保持一致
- 快反阈值根据实际业务需求调整
- 多时区配置需覆盖所有协作工厂所在时区
- ESG风险阻断需与合规团队确认触发规则
- 建议与 apparel-industry-data-planning 组合使用获取完整服装行业数据层
